# ABC 阶段判断与画图逻辑说明

这份文档只讨论 K 线工具里的 `ABC` 结构判断，以及页面上那张 `ABC 位置示意图` 是怎么生成的。

目标不是讨论 UI，而是把当前程序的判断方式拆开讲清楚，方便后续单独修规则。

---

## 1. 当前 ABC 判断在做什么

现在这套逻辑，并不是严格意义上去识别经典波浪理论里的 `A-B-C`。

它本质上是在做一件更工程化的事情：

- 结合当前位置
- 结合支撑 / 确认位 / 压力位
- 结合均线、动量、RSI
- 结合最近几根 K 线的抬高或走弱情况

来判断当前更像：

- `A 段`
- `B 段`
- `B→C 过渡`
- `C 段`
- 或者结构已经失败

对应代码入口：

- `donew/kline_robot_vercel/api/report.js`
- 函数：`summarizeAbcMomentum(...)`

---

## 2. 当前判断依赖哪些输入

当前 ABC 判断主要依赖以下变量：

- `support`：近期支撑位
- `pressure1`：近端确认位
- `pressure2`：更高一层压力
- `e20 / e60`：20 / 60 均线
- `mom10`：10 根动量
- `rsi14`
- 最近 4 根 K 线：
  - 收盘是否逐步抬高
  - 高点是否逐步抬高
- 最近 5 根量能和前 10 根量能对比

另外还会计算一个很关键的“相对位置”：

- `closePosition`
- `normalizedClosePosition`
- `confirmGapRatio`

其中最重要的是：

- `normalizedClosePosition`：当前价格在 `support -> pressure1` 这段区间里走到了哪里
- `confirmGapRatio`：距离确认位还有多远

---

## 3. 当前阶段判断顺序

当前逻辑是一个“从上到下命中即停”的状态机。

也就是说，谁写在前面，优先级就更高。

### 3.1 结构失败

如果满足：

- `last.close < support`
- 不在 `E20` 上方
- `momentum < 0`

则直接判成：

- `结构失败`
- `ABC 结构失效，优先按破位处理`

这类情况会直接跳出原 ABC 逻辑。

---

### 3.2 C 段延续

如果满足：

- `mom10 >= 3`
- 站上 `E20`
- 站上 `E60`
- 站上 `pressure2`

则判成：

- `C 段延续`
- `当前更像 C 段后半段 / 延续加速`

这代表已经不是普通回升，而是强延续。

---

### 3.3 C 段突破确认

如果满足：

- `mom10 >= 1`
- `aboveE20`
- `last.close >= pressure1`

则判成：

- `C 段突破确认`
- `当前更像 C 段突破确认 / 已站上确认位`

这是当前逻辑里“真正完成 C 段确认”的主要判断。

---

### 3.4 C 段上攻 / 冲击前高

如果满足：

- 站上 `E20`
- 最近几根 K 线持续抬高
- 当前价格已经比较靠近区间上部

则判成：

- `C 段上攻 / 冲击前高`

它的含义是：

> 当前位置已经脱离 C 点低位，正在向上修复，并尝试冲击原来的 B 高或前高。

---

### 3.5 C 段回升 / 靠近确认

如果满足：

- 站上 `E20`
- 最近趋势向上
- 已经靠近确认位

则判成：

- `C 段回升 / 靠近确认`

这类判断比“刚从 C 点抬头”更强，但又没有进入“正式突破确认”。

---

### 3.6 C 段止跌 / 初步回升

如果满足：

- 站上 `E20`
- 最近趋势刚转正
- 但价格位置还没有那么高

则判成：

- `C 段止跌 / 初步回升`

也就是：

> 更像从 C 点附近刚刚开始抬头，先看止跌是否成立，再看能否进一步走强。

---

### 3.7 C 段前确认

这是后面为了修正“靠近确认位却被误判成 B→C”新增的一层。

如果满足：

- `aboveE20`
- `recentTrend >= 0`
- `confirmGapRatio <= 0.14`

则判成：

- `C 段前确认`
- `当前更像 C 段前确认 / 靠近确认位`

它的含义是：

> 即使还没正式突破确认位，但已经非常接近确认位，应当归在 C 段前确认，而不只是普通过渡。

---

### 3.8 B→C 过渡

如果满足：

- 动量转正
- 站上 `E20`
- 但还没越过确认位

则判成：

- `B→C 过渡`
- `当前更像 B 末段，正在向 C 过渡`

这一步现在最容易和“C 段前确认”打架。

用户当前觉得不准，很多时候就是因为：

- 肉眼已经觉得接近确认位
- 但程序仍然停在 `B→C 过渡`

---

### 3.9 A / B 段其它状态

再往后才是：

- `A 段后回撤`
- `B 段健康回踩`
- `B 段偏弱整理`
- `A 段失败 / 转弱`
- `B 段止跌 / 等待 C`
- `C 段试探`

这些阶段更多是用来表达：

- 正在消化
- 还没形成明确 C 段
- 或者结构走弱

---

## 4. 当前图是怎么画出来的

页面中的 ABC 图不是根据真实 K 线重新拟合出一条波浪图。

它的实现方式是：

1. 先画一条固定的示意曲线
2. 再根据判断结果，把“当前点”放在曲线上的某个位置
3. 再把说明框贴到当前点附近

所以本质上它是：

> 标准示意图 + 当前阶段定位

而不是：

> 根据真实价格走势动态画出的 ABC 曲线

对应函数：

- `abcPositionSvg(abc)`

---

## 5. 图上的 A / B / C 三个点是固定模板点

当前图上的三个关键点并不是从真实价格里算出来的，而是固定模板坐标：

- A 点：左下
- B 点：上方高点
- C 点：右侧低点

颜色固定为：

- A：红
- B：绿
- C：蓝

这张图的任务是“表达阶段”，不是“复刻真实价格”。

---

## 6. 当前黄点是怎么放上去的

黄点的位置不是由真实末尾 K 线直接映射而来。

它主要由两个字段控制：

- `phaseKey`
- `progress`

### 6.1 phaseKey

它表示当前处于哪一大段：

- `A`
- `B`
- `BC`
- `C`
- `fail`
- `unknown`

### 6.2 progress

它是一个数值，表示在该阶段里走到了哪里。

例如当前代码里大概是这样：

- `A 段失败 / 转弱`：`0.18`
- `A 段后回撤`：`0.26`
- `B 段偏弱整理`：`0.44`
- `B 段健康回踩`：`0.48`
- `B 段止跌 / 等待 C`：`0.56`
- `B→C 过渡`：`0.58 ~ 0.70`
- `C 段止跌 / 初步回升`：`0.64+`
- `C 段回升 / 靠近确认`：`0.72+`
- `C 段前确认`：`0.74+`
- `C 段上攻 / 冲击前高`：`0.82+`
- `C 段突破确认`：`0.90+`
- `C 段延续`：`0.97`

也就是说：

> 先判断“属于哪个阶段”，再决定黄点落在模板曲线的哪个区间。

所以如果阶段判错了，图的位置一定也会跟着错。

---

## 7. 为什么现在会感觉位置判断不准

这部分是最关键的。

### 7.1 现在本质上不是“先找 A / B / C 点”

当前程序没有先去识别：

- 最近哪个低点是 A
- 最近哪个高点是 B
- 最近哪个回踩低点是 C

它是直接用：

- 当前价格相对支撑 / 确认位的位置
- 动量
- 均线
- 最近几根 K 线节奏

去估一个阶段。

所以它更像“状态机”，不是“波段分型器”。

---

### 7.2 B→C 和 C 前确认之间边界太粗

当价格已经比较接近确认位，但还没正式站上时，
肉眼常常会觉得：

- 已经在 C 段靠近确认位
- 甚至接近突破

但程序可能还停留在：

- `B→C 过渡`

于是文字和图的位置就显得偏后。

---

### 7.3 图用的是模板，不是实际走势映射

图上的曲线本身永远是固定形状。

所以即便文字判断对了，
只要真实 K 线的结构和模板差异较大，
用户还是会觉得：

- “实际走势更像在这个位置”
- “图却画在另一个位置”

---

## 8. 后续如果要改，建议怎么拆

建议把 ABC 判断拆成两层。

### 第一层：先找结构点

先识别真实波段里的：

- A 点候选：起涨低点 / 结构低点
- B 点候选：最近一波高点 / 峰值
- C 点候选：B 后回踩低点

也就是先把点找出来。

### 第二层：再判断当前位置

最后一根 K 线相对这些点的位置，再细分成：

- A 段启动
- B 段上冲
- B 段末端
- B→C 回踩
- C 附近止跌
- C 段回升
- 靠近确认位
- 突破确认位
- 突破后延续

这样图和文字会更贴近真实结构。

---

## 9. 当前最准确的理解方式

如果要用一句话总结当前实现：

> 这不是一个精确的波段点位识别器，而是一个基于关键位、动量、均线和最近 K 线节奏的 ABC 阶段状态机。

它适合回答：

- 当前更像哪一段
- 是启动、整理、过渡还是确认

但它不适合回答：

- 真实波段里的 A / B / C 点到底精确在哪里

---

## 10. 建议下次单独讨论的三个问题

为了把这块彻底讨论清楚，建议单独围绕这三个问题定规则：

1. `A / B / C 点到底怎么定义`
2. `当前位置阶段要拆成几档`
3. `图继续用模板图，还是改成真实走势映射图`

只要这三个问题定下来，后面的代码就容易稳定很多。
