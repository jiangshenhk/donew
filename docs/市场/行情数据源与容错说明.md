# 行情数据源与容错说明

## 目标

这份说明用于约束 `market-analysis-tool` / `/api/market-report` 的统一数据口径，避免行情源临时失败时把缺失值误当成 `0.00%` 输出。

## 当前执行口径

2026-07-11 起，市场分析工具改为分层取数：

1. 美股 ETF / 个股
   - 主用 `Nasdaq` 公共 quote + historical 接口
   - 目的：替代近期频繁返回 `Too Many Requests` 的 Yahoo Finance
2. 波动率 / 利率
   - 主用 `FRED` 官方公开 CSV
   - 当前已接入：
     - `VIXCLS` -> `VIX`
     - `DGS10` -> `10Y`
3. 数字货币
   - 当前仍尝试公开源
   - 如果源头不可达，页面必须输出“未取到”，不能写成 `0.00%`

## 容错规则

- 关键资产取不到时，报告中必须显示 `未取到`
- 不允许把缺失值当成 `0.00%`
- 报告顶部要输出 `数据告警`
- 风险判断允许在缺失数据下继续生成，但要明确提示“数据不完整”

## 当前已知情况

- `Yahoo Finance` 在当前环境下经常返回 `Too Many Requests`
- `Eastmoney` 历史 K 线接口在当前环境下也不稳定
- 因此：
  - 美股 ETF / 个股优先走 `Nasdaq`
  - 宏观日频优先走 `FRED`

## 默认源与降级顺序

### 1. 美股 ETF / 股票

- 默认主源：`Yahoo Finance quote + chart`
- 降级补取：`东方财富`
- 适用品种：`QQQ / SPY / IWM / QLD / TQQQ / SMH / SOXX / MAGS / EEM / FXI / KWEB / IBIT / MSTR / INTC / HOOD`

### 2. 宏观与跨资产指标

- 当前主源：`Yahoo Finance`
- 包括：`VIX / 10Y / DXY / USDJPY / WTI / 黄金`
- 这些品种如果抓取失败，当前版本不允许静默写成 `0.00%`，必须在报告中标记为 `未取到`

### 3. 新闻快讯

- 当前源：`金十财经搜索结果页 / xnews.jin10.com`
- 用途：补充最近 24 小时重大事件，不替代行情快照

## 容错规则

### 1. 失败值不是 0

以下情况统一视为无效数据：

- 最新价 `<= 0`
- 涨跌幅为空
- 涨跌幅绝对值异常大（当前阈值 `>= 95%`）
- 相对 20 日 / 50 日偏离异常大（当前阈值 `>= 95%`）

一旦命中无效条件：

- 页面和 Markdown 里显示 `未取到`
- 报告增加 `数据告警`
- 不允许再按中性、按 `0.00%`、或按“没有变化”解释

### 2. 关键指标缺失时的处理

关键指标包括：

- `QQQ / SPY / IWM`
- `SMH / SOXX`
- `BTC`
- `VIX / 10Y / DXY`

如果关键指标缺失：

- 报告顶部必须提示哪些指标未取到
- `数据来源` 一节必须列出实际来源、状态、备注
- 风险判断自动转保守，不把缺失当顺风

## 输出要求

报告必须同时输出：

1. `数据口径`
2. `数据告警`（如果存在）
3. `数据来源` 表格

这样做的目的只有一个：外部源临时失败时，用户看到的是“不完整，需要复核”，而不是被错误的 `0.00%` 误导。
