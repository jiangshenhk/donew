# MSTR 卖 Put 操作框架

> MSTR 不是普通股票，它本质上是“BTC beta + 公司资本结构 + 市场情绪”的高波动组合。因此，MSTR 卖 Put 不能只看权利金高低。

---

## 一句话框架

> **MSTR 只能在 BTC 主动走强、MSTR 没有明显弱于 BTC、市场风险偏好稳定时，用小仓、极远 OTM 的方式试探。**

---

## MSTR 的特殊风险

| 风险 | 含义 |
|---|---|
| BTC beta | BTC 小跌，MSTR 可能放大跌 |
| 公司溢价 | MSTR 相对 BTC 可能出现溢价压缩 |
| IV 高 | 权利金高，但尾部风险也高 |
| 跳空风险 | BTC 24小时交易，美股不开盘时仍会波动 |
| 流动性风险 | 快速下跌时 Put 价差可能变大 |

---

## 什么时候可以考虑卖？

| 条件 | 判断 |
|---|---|
| BTC 强于 Nasdaq | 必要条件 |
| BTC 站稳关键区间 | 必要条件 |
| MSTR 不弱于 BTC | 必要条件 |
| 纳指没有明显破位 | 辅助条件 |
| IV 足够高但非恐慌失控 | 辅助条件 |

---

## 什么时候不做？

- BTC 横盘或弱于美股；
- MSTR 高开冲高回落；
- Nasdaq 与 BTC 同时走弱；
- 10Y 快速上行；
- MSTR 财报、增发、融资、BTC重大事件附近；
- 只是因为权利金高而想卖。

---

## Strike 选择

| 市场状态 | Strike 选择 |
|---|---|
| BTC 强势突破 | 远 OTM，小仓 |
| BTC 稳但不强 | 极远 OTM 或不做 |
| BTC 弱 | 不做 |
| MSTR 弱于 BTC | 不做 |
| IV 很高但结构不好 | 不做 |

---

## 仓位原则

| 标的 | 仓位逻辑 |
|---|---|
| QLD | 可以作为主策略 |
| MSTR | 只能作为加速器 |

MSTR 仓位通常应明显小于 QLD。即使评级为 ✅，也不代表可以重仓。

---

## 执行清单

卖 MSTR Put 前至少检查：

- [ ] BTC 是否强于 Nasdaq；
- [ ] BTC 是否站稳关键价位；
- [ ] MSTR 是否没有弱于 BTC；
- [ ] 10Y 是否没有快速上冲；
- [ ] Nasdaq 是否没有明显破位；
- [ ] 权利金是否足够补偿尾部风险；
- [ ] 仓位是否小于 QLD 主仓。

---

## 核心纪律

> **MSTR 是加速器，不是底仓工具。**

做对时，它能提高收益；做错时，它会快速放大回撤。卖 MSTR Put 的核心不是“收高权利金”，而是只在 BTC 和市场结构同时配合时，承担小而可控的风险。
