# 投资策略_每日执行总控

> 本文件是 `docs/SellPut/策略/` 新架构下的每日投资决策入口。  
> 它不是单一“卖 Put 策略”，而是一个通用投资决策工作流：先判断市场环境，再选择标的，再调用策略模块，最后根据交易类型决定是否执行。

---

## 一、总控定位

本总控文件负责回答：

```text
今天能不能承担风险？
应该看哪些标的？
应该调用哪些策略？
最后适合哪一种动作：买入、定投、卖 Put、Covered Call、观察、暂停？
```

总控不直接替代策略文件。  
总控只做三件事：

1. 读取注意事项；
2. 读取标的池；
3. 调用策略模块并汇总成行动建议。

---

## 二、新架构原则

### 1. 标的池与交易动作分离

标的池只回答：

```text
看什么？
```

不回答：

```text
买不买？
卖不卖 Put？
卖不卖 Call？
```

一个标的可以同时服务多种交易动作：

| 标的 | 可用于 |
|---|---|
| QQQ / QLD | 市场风向、定投、卖 Put、风险偏好观察 |
| INTC | 个股趋势、半导体风向、卖 Put、买入观察 |
| MSTR / IBIT | BTC 风险偏好、卖 Put、仓位观察 |
| SMH / SOXX | AI / 半导体风向观察 |
| EEM | 新兴市场、美元流动性、亚洲风险偏好 |

---

### 2. 策略模块与交易动作分离

策略文件只回答某一类判断问题，不直接绑定交易动作。

| 策略 | 回答的问题 | 是否绑定卖 Put |
|---|---|---|
| 策略1：市场风向判断 | 当前环境是顺风、横风、逆风还是事件风 | 否 |
| 策略2：技术结构判断 | 标的是上涨、回调、破位、修复还是震荡；当前K线最像哪一种结构图 | 否 |
| 策略3：波动率与期权参数判断 | 当前波动率与期权价格是否补偿风险 | 仅涉及期权定价，不直接决定交易 |

最终是否卖 Put，由总控在汇总后决定。

---

### 3. 注意事项优先

任何交易动作前，必须先读取注意事项。

重点包括：

- 市场风险注意事项；
- 日元与日本利率风险注意事项；
- AI 交易拥挤度与接力真空注意事项；
- 期权卖方风险注意事项。

如果注意事项触发红灯，即使策略分数较高，也要降级。

---

## 三、每日执行流程

每日执行顺序如下：

```text
Step 1：读取注意事项
Step 2：读取标的池
Step 3：调用策略1，判断市场风向
Step 4：从标的池中选择当日相关标的
Step 5：调用策略2，判断技术结构
Step 6：如果涉及期权，再调用策略3
Step 7：根据交易类型输出行动建议
Step 8：记录失效条件与复盘要点
```

### Step 5 补充要求：策略2必须先定位

调用策略2时，必须优先读取并输出以下增强字段：

```text
当前位置
最像哪张结构图
123法则状态
2B法则状态
技术结构等级
是否进入下一步交易参数分析
```

策略2的核心不是直接决定交易，而是先回答：

```text
当前K线在结构里处于什么位置？
它最像哪一张标准结构图？
下一步向上升级、向下降级、结构失效分别看哪里？
```

常用当前位置标签包括：

```text
A点附近
A点后弱弹
B点压力区
B到C回踩途中
C点观察区
C点确认转强
突破B
破位反抽
底部2B
顶部2B
强趋势回踩
结构不清
```

总控只使用策略2的技术结构结论作为输入，不允许策略2直接给出最终交易动作。

---

## 四、交易动作分类

本总控最终输出的动作必须属于以下几类：

| 动作 | 含义 |
|---|---|
| 买入 | 适合现货建仓或加仓 |
| 定投 | 按长期计划执行，不因短期波动改变核心节奏 |
| 卖 Put | 市场、技术、波动率和接货意愿同时合格 |
| Covered Call | 已持仓标的上涨过快或进入压力区时考虑 |
| 观察 | 条件不完整，等待确认 |
| 暂停 | 风险过高，不执行新动作 |

## 六、总控评分规则

最终等级取以下模块中的最低等级：

```text
最终等级 = 市场风向等级、注意事项等级、技术结构等级、交易参数等级中的最低等级
```

| 等级 | 含义 | 动作 |
|---|---|---|
| A | 顺风，条件完整 | 可执行 |
| B | 横风或轻微风险 | 小仓、远离风险区 |
| C | 风险偏高或条件不足 | 观察 |
| D | 逆风、事件风、结构破坏 | 暂停 |

### 策略2技术结构等级的使用方式

策略2输出的 `当前位置`、`结构图匹配`、`123法则状态`、`2B法则状态` 用于解释技术结构等级，但最终总控只把策略2归纳成 A / B / C / D 参与总控评分。

| 策略2增强字段 | 总控中的用途 |
|---|---|
| 当前位置 | 判断当前是机会区、观察区还是危险区 |
| 最像哪张结构图 | 判断主路径和备选路径 |
| 123法则状态 | 判断趋势是否确认反转或确认转弱 |
| 2B法则状态 | 判断是否出现假突破/假跌破预警 |
| 技术结构等级 | 参与总控最低等级评分 |
| 是否进入下一步交易参数分析 | 决定是否调用策略3 |

兼容说明：

```text
外部系统如果只支持旧版字段，可继续读取技术结构等级 A / B / C / D。
新增字段属于增强解释字段，不改变原有等级含义。
程序调用不要依赖章节编号，应优先依赖字段名。
```

## 七、交易类型与调用逻辑

### 1. 买入 / 加仓

必须调用：

```text
策略1：市场风向判断
策略2：技术结构判断
```

不强制调用策略3。

---

### 2. 定投

必须调用：

```text
策略1：市场风向判断
```

策略2用于判断是否正常定投、加码、减速或暂停。

---

### 3. 卖 Put

必须调用：

```text
策略1：市场风向判断
策略2：技术结构判断
策略3：波动率与期权参数判断
期权卖方风险注意事项
```

只有当以下条件同时满足，才允许进入候选：

```text
市场允许 + 技术结构健康 + 波动率补偿合理 + 自己愿意接货
```

卖 Put 对策略2的额外要求：

| 策略2位置 | 总控默认处理 |
|---|---|
| C点确认转强 | 可进入策略3参数分析 |
| 强趋势回踩 | 可进入策略3参数分析 |
| 底部2B观察 | 先观察，除非市场风向和波动率补偿都很好 |
| C点观察区 | 可准备，但不直接执行 |
| A点附近 | 不主动执行 |
| A点后弱弹 | 不主动执行近价Put |
| B到C回踩途中 | 不猜底，不主动执行 |
| 破位反抽 | 暂停 |
| 顶部2B | 降级或暂停 |
| 结构失败 | 暂停 |

核心纪律：

```text
卖 Put 最好的技术候选区，通常不是A点，也不是B到C下跌途中，
而是C点确认转强，或强趋势回踩确认不破。
```

---

### 4. Covered Call

必须调用：

```text
策略1：市场风向判断
策略2：技术结构判断
策略3：波动率与期权参数判断
```

重点判断：是否处于压力区、是否过热、Call 权利金是否补偿被行权风险。

Covered Call 对策略2的额外关注：

| 策略2位置 | Covered Call含义 |
|---|---|
| B点压力区 | 可关注Call权利金 |
| 突破B后短线过热 | 可关注分批Covered Call |
| 顶部2B | 优先考虑防守性Covered Call或减仓 |
| 高位ABC转弱 | 避免只靠Covered Call防守，需评估现货风险 |

## 八、每日标准输出

每次日报必须包含：

1. 今日市场总览；
2. 注意事项触发情况；
3. 市场风向等级；
4. 标的池分组观察；
5. 技术结构判断；
6. 如涉及期权，补充波动率与期权参数；
7. 最终交易动作；
8. 失效条件；
9. 明日复盘重点。

### 技术结构判断的最低输出要求

第5项“技术结构判断”必须至少包含：

```text
当前位置
最像哪张结构图
123法则状态
2B法则状态
关键支撑
关键压力
失效条件
技术结构等级
是否进入下一步交易参数分析
```

如果是单个标的分析，建议额外画出文字结构图：

```text
A点 / B点 / C点 / 当前价 / 升级路径 / 降级路径
```

例如：

```text
当前最像：图3｜低位ABC的C点观察区

          B（压力区）
         /\
        /  \
       /    \__ 当前价
A_____/        C？

升级：站回短均线并突破B
降级：跌破A且反抽站不回
```

## 九、核心纪律

```text
标的池只决定看什么。
策略只决定怎么看。
总控才决定做什么。
```

不因为某个 Put 权利金突然变高就交易。  
不因为某个标的是 AI 就自动提高权重。  
不因为长期看好就忽略短期风向。  
不因为策略结果好就跳过注意事项。

新增结构纪律：

```text
先定位，再判断；
先判断结构图，再判断指标；
先判断是否进入下一步参数分析，再决定交易动作。
```

---

## 十、一句话原则

```text
先判断环境，再判断结构，最后才判断价格和交易动作。
```

```text
策略2负责定位当前K线在哪张结构图上；
总控负责决定这张结构图是否允许进入交易。
```
