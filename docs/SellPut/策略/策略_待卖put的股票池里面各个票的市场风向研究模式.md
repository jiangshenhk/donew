## 一、最高原则

### 1. 风向优先于安全垫

7天内卖 put，最大风险不是估值错，而是短期风向突变：

- 指数突然 risk-off
- VIX 跳升
- 10Y / DXY / 油价冲击
- BTC 跳水
- 半导体退潮
- 个股事件跳空
- 盘后无法止损

所以必须先判断：

> 当前是顺风、横风、逆风，还是事件风？

安全垫只在慢跌里有效。  
如果风向逆转，安全垫会迅速失效。

### 2. 先做市场风向过滤，再看标的

固定顺序：

```text
短期风向灯
↓
事件风险过滤
↓
市场宽度与波动率确认
↓
标的相对强弱
↓
期权补偿是否足够
↓
卖 put 执行等级
```

不能直接从“QLD / EEM / MSTR / INTC 价格便宜不便宜”开始。

### 3. 只关注 1–7 个交易日风险

所有判断必须服务短线卖 put：

- 不讨论长期估值太多
- 不用长期看好替代短期风控
- 不因为长期愿意持有，就忽略短线跳空风险
- 不因为权利金高，就忽略真实风险

## 二、短期风向灯

每天必须先输出市场风向灯。

| 风向 | 定义 | 卖 put 动作 |
|:---|:---|:---|
| 🟢 顺风 | 指数稳，VIX稳定或回落，10Y/DXY/油价不冲击，市场宽度健康，主线有扩散 | 可以卖7天put |
| 🟡 横风 | 指数高位震荡，VIX小升，10Y接近压力，小盘弱，AI分化 | 只能极远OTM、小仓 |
| 🔴 逆风 | QQQ/SPY破位，VIX跳升，10Y/DXY/油价冲击，半导体转弱，BTC弱 | 暂停卖put |
| ⚫ 事件风 | CPI/FOMC/非农/财报/重大地缘/标的事件临近 | 禁止新增或只观察 |

必须给一句：

> **今日短期风向：🟢 / 🟡 / 🔴 / ⚫**

并说明为什么。

## 四、短线卖 Put 环境评分

每天必须打分：

| 项目 | 评分 | 判断 |
|:---|:---|:---|
| **短期风向稳定度** | x / 10 | 是否适合承担7天风险 |
| **波动率风险** | x / 10 | VIX / IV 是否可能突升 |
| **市场宽度** | x / 10 | 是否只有少数股票硬撑 |
| **宏观冲击风险** | x / 10 | 利率、美元、日元、油价是否冲击 |
| **事件跳空风险** | x / 10 | 是否临近CPI/FOMC/财报/重大事件 |
| **期权卖方友好度** | x / 10 | 权利金是否补偿风险 |

最后给：

| 总结项 | 判断 |
|:---|:---|
| **7天卖put环境** | 适合 / 谨慎 / 不适合 |
| **整体执行等级** | A / B / C / D |

执行等级：

| 等级 | 动作 |
|:---|:---|
| **A** | 可以正常卖put |
| **B** | 可以小仓远OTM |
| **C** | 只观察，等IV抬升或回调 |
| **D** | 禁止新增 |

## 六、事件风险过滤器

7天卖put前必须检查未来7天：

| 事件 | 是否存在 | 处理 |
|:---|:---|:---|
| CPI / PPI / PCE | 是 / 否 | 高风险日前不卖贴价put |
| FOMC / Fed讲话 | 是 / 否 | 降低Delta或暂停 |
| 非农 / 失业率 | 是 / 否 | 数据日前谨慎 |
| 目标标的财报 | 是 / 否 | 原则上不跨财报 |
| 评级 / 监管 / 政策事件 | 是 / 否 | 有则降级 |
| 周末BTC风险 | 是 / 否 | MSTR避免跨周高风险仓 |
| 地缘冲突headline | 是 / 否 | 油价/VIX冲击时暂停 |
| BOJ会议 / 日本CPI / 汇率干预风险 | 是 / 否 | 有则降低Delta，日元急升时暂停高beta卖put |

必须输出：

> **事件风险：低 / 中 / 高。**

如果事件风险高，整体执行等级最高只能是 C，通常为 D。

## 七、期权执行硬校验

如果有实时期权链数据，必须检查：

| 检查项 | 要求 |
|:---|:---|
| **Delta** | 是否符合当前风向 |

如果没有实时期权链数据，必须写：

> 期权链数据不足，以下Delta与执行区间只是风控框架，实际下单前必须复核实时IV、bid-ask、OI、事件日历和breakeven。

## 八、止损 / 降级规则

7天卖put必须提前写好止损和降级条件。

| 触发条件 | 动作 |
|:---|:---|
| Put价格变为开仓价2倍 | 重新评估，通常止损或滚动 |
| Delta从0.08升到0.20以上 | 不再视为远OTM，必须处理 |
| VIX单日跳升10%以上 | 暂停新增 |
| QQQ跌破5日/10日趋势 | QLD降级 |
| 10Y快速上行并压制科技 | QLD降级或暂停 |
| USDJPY快速下行、日元急升 | 视为carry trade平仓风险，整体环境至少降一级 |
| USDJPY快速上行且DXY/10Y同步上行 | 视为美元流动性冲击，QLD降级，MSTR必须等待BTC确认 |
| 日本10Y/30Y快速上行 | 检查是否外溢冲击美债，成长股和高beta标的降级 |
| BTC跌破关键位 | MSTR暂停 |
| MSTR弱于BTC | MSTR暂停 |
| INTC弱于SOXX且放量 | INTC暂停 |
| 出现标的独立利空 | 对应标的暂停 |
| 事件日前权利金没有明显补偿 | 不开仓 |

## 九、最终输出格式

每日正式输出，必须优先使用“市场结构日报 / 周报”风格来承载下面的规则，而不是只做机械表格填空。

输出时，优先参考以下历史文档的结构、语气和层次：

- `docs/市场/2026-04-12市场结构日报(晚8点).md`
- `docs/市场/2026-06-03市场结构日报(晚8点).md`
- `docs/市场/2026-06-06市场结构周报.md`

要求：

- 先参考这些文档的“日报 / 周报写法”，包括一句话结论、总览表、资金流向异动、策略矩阵、跨资产联动、驱动拆解、下一步看点、动作清单。
- 再把本文件里的风向灯、事件过滤、环境评分、标的执行等级、最终执行表嵌进去。
- 不允许只输出机械化的模板答题；必须尽量写成可直接用于盘前或盘后决策的市场结构日报。

### 产出物类型

本策略的正式输出，必须能单独保存为 Markdown 文件并上传到 Git。  
因此每次输出前，必须先明确自己在写哪一种：

1. `日报（早八点）`
   - 用途：盘前预案
   - 时间口径：基于最近一个完整美股交易日收盘、盘后消息、亚洲/欧洲早盘与跨资产最新数据
   - 标题风格参考：`2026-06-20市场结构日报(早8点).md`
   - 重点：今天开盘前的风险偏好、先看什么、哪些标的可以进入候选

2. `日报（晚八点）`
   - 用途：盘后复盘 + 次日预案
   - 时间口径：基于当日美股收盘、盘后消息与跨资产数据
   - 标题风格参考：`2026-06-20市场结构日报(晚8点).md`
   - 重点：当天真正交易了什么、结构是否变化、次日该升仓还是降级

3. `周报`
   - 用途：周度结构复盘 + 下周预案
   - 时间口径：基于周五收盘、周末新闻与跨资产周末数据
   - 标题风格参考：`2026-06-20市场结构周报.md`
   - 重点：本周真正发生的结构性变化、资金流向迁移、下周核心观察点

### 输出要求

- 输出必须默认写成可直接保存到 Git 仓库的 Markdown 成稿，而不是聊天式摘要。
- 标题必须带日期和类型，例如：`# 📊 市场结构日报` 或 `# 📊 市场结构周报`
- 标题下一行必须写清楚时间口径，例如：  
  `**2026年6月20日｜香港时间周六晚8点（基于截至6月19日最近一个完整美股交易日收盘与跨资产数据）**`
- 内容必须能单独成文，不依赖聊天上下文。

推荐顺序固定为：

1. 标题与时间口径  
   例如：`2026年6月20日｜基于美股最近一个完整交易日收盘与最新跨资产数据`

2. 一句话结论  
   必须先用交易语言给出一句话结论，直接回答今天适不适合卖put、主策略是谁、谁降级。

3. 总览表  
   必须先给这张总览表：

|:---|:---|

4. 资金流向异动

| 异动 | 变化 | 指标 | 信号解读 |
|:---|:---|:---|:---|

   公共股票池的每日必分析标的，以及必要的 `QQQ / SPY / BTC` 等风向观察标的。

6. 资金流向与资产联动  
   尽量拆成以下三块：
   - 跨资产资金流向
   - 关键联动
   - 谁在说真话

7. 宏观资产数据  
   必须覆盖最重要的宏观方向盘，优先包括：
   `DXY`、`美2Y / 10Y / 30Y`、`Japan 10Y / 30Y`、`USDJPY`、`Gold`、`WTI / Brent`、`BTC`、`VIX`

8. 风险资产表现  
   必须至少覆盖：
   公共股票池的每日必分析标的；禁用清单中的标的不分析为交易候选。

9. 驱动拆解  
   尽量用简洁的因果链表示，例如：

```text

    ↓

    ↓
```

10. 下一个观察点  
    用表格列出最重要的 4–10 个观察点，并说明其交易意义。

## 十、写作风格

必须：

- 结论前置
- 交易语言
- 少新闻复述
- 多价格验证
- 多反证
- 多失效条件
- 不写宏观八股文
- 不把长期看好当短线卖put理由
- 不把安全垫当作风控核心

最终目标：

> 帮我判断今天市场短期风向是否允许卖 put，并按公共股票池的每日必分析池输出结构、参数与账户风险结论；禁用标的不得给出交易建议。
