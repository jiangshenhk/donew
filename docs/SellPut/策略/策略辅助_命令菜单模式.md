# 策略辅助_命令菜单模式

> 用途：本文件定义 Sell Put 策略文档组的交互入口。任何读取本策略目录的 AI 助手，在用户输入“菜单”或“命令菜单”时，都应先输出本文件的菜单选项，等待用户选择后再执行对应动作。

---

## 一、触发词

当用户输入以下任意内容时，进入命令菜单模式：

```text
菜单
命令菜单
Sell Put 菜单
卖 Put 菜单
```

不要在触发菜单后立刻跑数据、分析标的或修改文件。必须先输出菜单，让用户选择。

---

## 二、默认菜单

输出格式固定为：

```text
请选择要执行的操作：

1. 跑今天的 Sell Put 总控
2. 跑今天的 Sell Put 总控（不重复获取数据，使用本地数据）
3. 跑 Delta 0.12-0.18 自动提醒，获取适合标的信息，并列出年化和安全垫
4. 分析某个标的是否适合卖 Put
5. 输入某个股票，分析动量和 ABC 模式
6. 单独分析今天市场情况
7. 查看 / 更新模拟盘
8. 更新策略文档并提交 Git
```

如果用户只回复数字，按对应编号执行。

如果用户回复“数字 + 标的”，例如：

```text
4 EEM
4 MSTR
4 HOOD
5 INTC
5 HOOD
6
7
8
```

则直接执行对应编号，并把后面的标的作为分析对象。

---

## 三、分级数据获取原则

不同菜单只能获取完成该动作所需的最小数据集，不能默认跑全量数据。

| 菜单 | 数据范围 | 允许获取 | 禁止自动获取 |
|---|---|---|---|
| 1. 跑今天的 Sell Put 总控 | 全量数据 | 市场快照、Barchart 期权概览、Nasdaq 期权链、Delta 提醒、金十市场新闻、模拟盘读取 | 不禁止；这是唯一默认全量入口 |
| 2. 本地数据跑总控 | 本地全量数据 | 只读取当天已存在的全量数据文件 | 不重新请求外部数据 |
| 3. Delta 0.12-0.18 自动提醒 | 期权筛选数据 | Barchart 期权概览、Nasdaq 期权链、Delta 提醒 | 金十新闻、市场总览日报、模拟盘写入 |
| 4. 分析某个标的是否适合卖 Put | 单标的决策数据 | 该标的市场风向、技术快照、必要期权参数 | 全股票池期权链、无关标的新闻 |
| 5. 输入某个股票，分析动量和 ABC 模式 | 单标的技术数据 | 该标的 OHLCV、均线、近期价格结构 | 期权链、Delta、Barchart 期权概览、金十新闻、模拟盘 |
| 6. 单独分析今天市场情况 | 市场风向数据 | 指数、VIX、DXY、10Y、USDJPY、WTI、黄金、BTC/IBIT/MSTR、SMH/SOXX、EEM/FXI/KWEB | 期权链、Delta、Barchart 期权概览、模拟盘 |
| 7. 查看 / 更新模拟盘 | 模拟盘数据 | 模拟盘文件；只有用户要求记录时读取候选 CSV | 市场行情、期权链 |
| 8. 更新策略文档并提交 Git | Git/文档数据 | 需要修改的策略文件和 Git 状态 | 行情数据、期权链 |

原则：

```text
第 1 项 = 全量入口；
第 3 / 5 / 6 项 = 各自只抓自己的必要数据；
没有用户明确要求，不得用第 1 项代替第 3 / 5 / 6 项。
```

## 四、各菜单动作定义

### 1. 跑今天的 Sell Put 总控

执行每日入口脚本：

```powershell
cd D:\codex\金融
powershell -ExecutionPolicy Bypass -File .\donew\docs\SellPut\策略\run_daily_sell_put_data.ps1
```

然后按总控顺序输出：

```text
市场风向
-> 日元套息风险
-> 个股技术结构 / ABC
-> 期权链参数
-> 账户风险
-> 结论：适合 / 小仓适合 / 不适合
```

必须引用以下自动生成文件：

```text
D:\codex\金融\outputs\daily_sell_put_data_pack_YYYY-MM-DD.md
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.md
D:\codex\金融\donew\docs\SellPut\模拟盘.md
```

### 3. 跑 Delta 0.12-0.18 自动提醒，获取适合标的信息，并列出年化和安全垫

如果当天数据已经存在，读取：

```text
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.md
```

如果当天数据不存在，执行第 3 项专用轻量脚本，不执行每日总控入口。

执行时只允许抓取期权筛选所需数据，不允许抓取金十新闻，不允许生成市场总览日报，不允许写入模拟盘。

默认执行：

```powershell
cd D:\codex\金融\donew
powershell -ExecutionPolicy Bypass -File .\scripts\run_delta_alert_data.ps1
```

输出时必须列出：

```text
Symbol / Expiry / DTE / Strike / Bid / Ask / Mid / Est Delta / Safety / Mid Annual / OI / Volume
```

必须明确列出每个候选的 `Mid Annual` 年化和 `Safety` 安全垫；估算 Delta 只用于初筛，真实交易前必须用富途复核真实 Delta 和实时 Bid/Ask。

### 4. 分析某个标的是否适合卖 Put

输入格式：

```text
4 标的
分析 标的
```

输出顺序固定为：

```text
市场环境
技术结构
期权参数
账户与仓位风险
结论：适合 / 小仓适合 / 不适合
```

如果标的是强势股，必须额外读取：

```text
策略_强势股ABC上涨结构下的SellPut模式.md
策略_强势股ABC模块_总控接入规则.md
```

不要只因为 RSI 高、新高、IV 高就否决；必须先判断是否属于“强势股第一波上涨后，B 段整理，再向上 C 段”的多头 ABC 结构。

### 7. 查看 / 更新模拟盘

默认读取：

```text
D:\codex\金融\donew\docs\SellPut\模拟盘.md
```

可执行动作：

```text
7 查看
7 记录今天第一条候选
7 平仓 标的 到期日 行权价 买回价
```

如果是“记录今天第一条候选”，执行：

```powershell
cd D:\codex\金融\donew
powershell -ExecutionPolicy Bypass -File .\scripts\record_sell_put_paper_trade.ps1 -AlertCsvFile D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.csv
```

模拟盘只记录纸上交易，不代表真实下单指令。

### 8. 更新策略文档并提交 Git

先确认需要修改的文档范围。默认只修改：

```text
D:\codex\金融\donew\docs\SellPut\策略\
D:\codex\金融\donew\docs\SellPut\模拟盘.md
D:\codex\金融\donew\scripts\
```

修改完成后执行：

```powershell
cd D:\codex\金融\donew
git status --short
git add -- <changed files>
git commit -m "<summary>"
git push origin main
```

如果网络导致 push 失败，必须明确说明：

```text
本地 commit 已完成，但 GitHub push 失败；
当前 main 领先 origin/main 几个 commit；
待网络恢复后执行 git push origin main。
```

### 5. 输入某个股票，分析动量和 ABC 模式

输入格式：

```text
5 标的
动量 ABC 标的
ABC 标的
```

该菜单只做技术结构判断，不直接输出卖 Put 结论，也不自动筛期权链。输出顺序固定为：

默认只抓取该标的技术数据：

```powershell
cd D:\codex\金融\donew
powershell -ExecutionPolicy Bypass -File .\scripts\fetch_symbol_technical_snapshot.ps1 -Symbol <标的>
```

该命令只生成：

```text
D:\codex\金融\outputs\technical_snapshot_<标的>_YYYY-MM-DD.csv
D:\codex\金融\outputs\technical_snapshot_<标的>_YYYY-MM-DD.md
```

第 5 项不得自动抓期权链、Delta 提醒、Barchart 期权概览、金十新闻或模拟盘。

```text
1. 当前趋势位置
2. 动量状态
3. ABC 结构草图
4. ABC 结构判断
5. 强势股多头 ABC 或高位向下 ABC 的分歧点
6. 关键价位：A 高点 / B 低点 / C 确认位 / 结构失效位
7. 对 Sell Put 的影响：可进入期权筛选 / 只能观察 / 技术上不适合
```

`ABC 结构草图` 必须用 ASCII 图画出 A 高点、B 低点、C 确认位和结构失效位；不能只用文字描述。

必须读取：

```text
策略_待卖put的股票池里面各个票根据动量原理和abc结构判断个股的策略.md
```

如果标的是强势股或近期大涨股，还必须读取：

```text
策略_强势股ABC上涨结构下的SellPut模式.md
策略_强势股ABC模块_总控接入规则.md
```

判断规则：

1. 不要把“第一波上涨后的正常回调”直接判为顶部。
2. 先检查是否属于强势股多头 ABC：A 段放量上涨，B 段缩量整理守住关键支撑，C 段重新走强。
3. 只有出现 `C 点低于 A 点` 并有效跌破 `B 点`，才按高位向下 ABC 或顶部结构处理。
4. 如果处在 B 段整理中，结论最多为“观察，等待 C 段确认”，不因为权利金高而直接进入 Sell Put。
5. 如果结构确认向上，再交给第 3 项或第 4 项检查 Delta、DTE、年化、安全垫和账户风险。

### 6. 单独分析今天市场情况

输入格式：

```text
6
市场
今天市场
单独分析市场
```

该菜单只做市场风向判断，不直接分析单个股票，不输出具体 Strike，不自动给卖 Put 下单结论。

执行时只允许抓取关键市场行情，不允许执行完整 Sell Put 数据入口，不允许抓取期权链，不允许生成 Delta 提醒，不允许写入模拟盘。

默认执行：

```powershell
cd D:\codex\金融\donew
powershell -ExecutionPolicy Bypass -File .\scripts\fetch_market_snapshot.ps1
```

该命令只生成：

```text
D:\codex\金融\outputs\market_snapshot_YYYY-MM-DD.csv
D:\codex\金融\outputs\market_snapshot_YYYY-MM-DD.md
```

第 6 项不得为了市场分析而自动生成以下文件：

```text
D:\codex\金融\outputs\nasdaq_put_chain_YYYY-MM-DD.csv
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.csv
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.md
```

输出顺序固定为：

```text
1. 美股大盘状态：QQQ / SPY / IWM / VIX
2. 高 beta 与科技风险：QLD / TQQQ / SMH / SOXX / 七姐妹
3. 日元套息风险：USDJPY / 日本利率 / 日元是否快速升值
4. 美元与利率：DXY / 10Y 美债 / 流动性压力
5. 商品与地缘：WTI / 黄金 / 中东或突发事件
6. Crypto 链条：BTC / IBIT / MSTR 风险传导
7. 今日 Sell Put 环境评级：顺风 / 中性 / 逆风
8. 对今日仓位的影响：可正常筛选 / 只做小仓 / 只观察
```

必须读取：

```text
策略_待卖put的股票池里面各个票的市场风向研究模式.md
日报周报/策略_每日市场判断怎么看GPT提示词_资金流向情况.md
```

如果当天已经存在自动数据包，优先读取：

```text
D:\codex\金融\outputs\market_snapshot_YYYY-MM-DD.md
D:\codex\金融\outputs\market_snapshot_YYYY-MM-DD.csv
```

如果当天市场快照不存在，直接执行 `scripts\fetch_market_snapshot.ps1`。不要询问是否执行每日入口脚本；第 6 项不是总控，不需要期权链、Delta、Barchart 期权概览或模拟盘数据。

评级规则：

```text
顺风：指数趋势健康，VIX稳定或下降，日元套息无升温，信用/利率/美元无异常，热点资产没有系统性破位。
中性：指数分化或短线回调，VIX温和上升，局部资产承压；只允许严格按 Delta、DTE、年化、安全垫筛选。
逆风：VIX快速上升、QQQ/高 beta 明显弱于 SPY、日元快速升值、BTC/IBIT/MSTR 链条破位、地缘或利率冲击未落地；原则上不新开高 beta Put。
```

输出结论必须使用：

```text
今日市场：顺风 / 中性 / 逆风
Sell Put 动作：正常筛选 / 小仓筛选 / 暂停新仓
重点风险提醒：...
```

### 2. 跑今天的 Sell Put 总控（不重复获取数据，使用本地数据）

输入格式：

```text
2
本地总控
不抓数据跑总控
使用本地数据跑总控
```

该菜单用于当天已经抓过数据、不希望重复请求 Barchart、Nasdaq、金十时使用。执行每日入口脚本，但必须带 `-UseExistingData`：

```powershell
cd D:\codex\金融
powershell -ExecutionPolicy Bypass -File .\donew\docs\SellPut\策略\run_daily_sell_put_data.ps1 -UseExistingData
```

该模式只读取本地今天已有文件：

```text
D:\codex\金融\outputs\barchart_options_overview_YYYY-MM-DD.csv
D:\codex\金融\outputs\nasdaq_put_chain_YYYY-MM-DD.csv
D:\codex\金融\outputs\jin10_market_news_YYYY-MM-DD.csv
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.csv
D:\codex\金融\outputs\sell_put_delta_alerts_YYYY-MM-DD.md
```

如果 `barchart_options_overview_YYYY-MM-DD.csv` 或 `nasdaq_put_chain_YYYY-MM-DD.csv` 不存在，必须提示用户先执行第 1 项抓取数据，不得静默使用旧日期数据。

该模式不重复写模拟盘。输出顺序与第 1 项一致：

```text
市场风向
-> 日元套息风险
-> 个股技术结构 / ABC
-> 期权链参数
-> 账户风险
-> 结论：适合 / 小仓适合 / 不适合
```

## 四、安全边界

1. 菜单命令不得直接触发真实下单。
2. 所有“卖出 Put”动作默认都是模拟或研究，除非用户明确说“我已经在富途手动成交了，帮我记录”。
3. 真实交易前必须用券商实时盘口复核：正股价格、Bid/Ask、真实 Delta、OI、成交量、保证金和事件风险。
4. CRCL 禁止交易，任何菜单动作都不得把 CRCL 纳入候选。
5. 如果数据缺失或期权链字段不完整，结论降级为“观察 / 不适合”，不得强行给具体下单价。
