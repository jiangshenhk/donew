> 本文优先服务于程序判断规范。新增内容应尽量写成“可判断、可拆细、可画图、可输出”的规则，而不是只写概念。

> 本策略只判断标的的技术结构，不绑定买入、卖 Put、卖 Call 或其他交易动作。  
> 它只回答：当前标的是上涨、回调、修复、转弱、破位，还是震荡？  
> 单个标的技术结构判断默认采用“文字说明 + 结构图”双输出。

## 一、主判断入口：2B、123与ABC的优先顺序

程序不能先问“像不像ABC”，而应先判断当前走势最适合哪一种结构语言。

```text
第一步：检查是否存在2B假突破或假跌破
第二步：检查是否正在形成或完成123趋势转折
第三步：检查是否具有清晰的ABC三点结构
第四步：三者都不清晰时，输出“不适合判断”
```

三种工具的分工：

| 方法 | 主要作用 | 特点 |
|---|---|---|
| 2B法则 | 提前发现假突破或假跌破 | 最灵敏，但假信号较多 |
| 123法则 | 判断原趋势是否真正扭转 | 更慢、更稳健 |
| ABC结构 | 标出价格当前所处位置和演化路径 | 最直观，适合图形化表达 |

核心原则：

```text
2B负责预警，123负责确认，ABC负责定位。
均线、动量、成交量负责验证结构质量。
```

### 1. 优先归类规则

| 当前走势特征 | 主判断 | 说明 |
|---|---|---|
| 突破前高后快速跌回，或跌破前低后快速收回 | 2B | 先判断真假突破，不急着套ABC |
| 趋势线被破坏，随后出现不创新高/低，并再次穿越关键前低/高 | 123 | 属于趋势转折过程 |
| 已有明确A、B、C三点，且价格处于A→B、B→C、C确认、突破B或跌破B等位置 | ABC | 可进入标准ABC图 |
| 只有局部反弹或回踩，前高前低不明确 | 不适合判断 | 不强行归类 |
| 强趋势中的正常均线回踩 | 强趋势回踩 | 不强行解释成低位ABC |
| 跌破关键支撑后反抽但无法站回 | 破位反抽 | 不强行解释成C点 |
| 长时间在清晰上下沿之间运行 | 箱体震荡 | 等待箱体突破或跌破 |

### 2. 主判断与结构演化必须分开

同一段行情可能依次经历：

```text
底部2B → 下跌转强123 → 低位ABC确认
顶部2B → 上涨转弱123 → 高位ABC确认
```

因此程序必须同时保留两个字段：

```text
主判断类型：当前最应使用的判断模型
结构演化背景：此前出现过什么结构，后续可能演化为什么结构
```

例如：

```text
主判断：123
结构演化背景：顶部2B之后
当前阶段：上涨转弱123第二步
```

不能因为较早出现过2B，就忽略行情已经发展到更后面的123或ABC阶段。

## 二、123法则的程序判断规范

### 1. 上涨转弱

| 状态 | 必要条件 | 技术含义 | 输出方向 |
|---|---|---|---|
| 上涨未破坏 | 趋势线有效，仍能创新高 | 主升结构仍在 | 偏多 |
| 123第一步 | 上升趋势线被跌破 | 上涨节奏首次破坏 | 中性偏弱 |
| 123第二步 | 反弹不再创新高 | 多头动能衰减 | 偏空预警 |
| 123第三步确认 | 再次跌破前低 | 上涨转弱正式确认 | 偏空 |

程序不能跳步：

```text
只跌破趋势线时，只能输出“123第一步观察”。
没有出现反弹不创新高和跌破前低，不能直接判断顶部成立。
```

### 2. 下跌转强

| 状态 | 必要条件 | 技术含义 | 输出方向 |
|---|---|---|---|
| 下跌未破坏 | 下降趋势线有效，仍创新低 | 主跌结构仍在 | 偏空 |
| 123第一步 | 下降趋势线被突破 | 下跌节奏首次破坏 | 中性偏强 |
| 123第二步 | 回踩不再创新低 | 空头力量衰减 | 修复候选 |
| 123第三步确认 | 再次突破前高 | 下跌转强正式确认 | 偏多 |

```text
只有止跌反弹时，只能写“下跌转强观察”。
没有回踩不创新低和突破前高，不能直接判断反转成立。
```

## 三、2B法则的程序判断规范

### 1. 底部2B

| 状态 | 必要条件 | 技术含义 | 输出方向 |
|---|---|---|---|
| 前低下破 | 跌破旧低 | 市场尝试继续杀跌 | 偏空 |
| 2B观察 | 跌破后快速收回前低上方 | 假跌破候选 | 中性偏强 |
| 2B成立 | 收回后不再快速失守，并站稳短线关键位 | 空头衰减 | 偏多观察 |
| 2B失败 | 收回后再次失守前低 | 主跌延续 | 偏空 |

```text
底部2B只说明假跌破具有观察价值，
不代表已经形成低位ABC，也不代表上涨趋势已经成立。
```

### 2. 顶部2B

| 状态 | 必要条件 | 技术含义 | 输出方向 |
|---|---|---|---|
| 前高突破 | 上破旧高 | 市场尝试继续冲高 | 偏多观察 |
| 2B观察 | 突破后快速跌回前高下方 | 假突破候选 | 中性偏弱 |
| 2B成立 | 跌回后无法重新站上前高，并开始转弱 | 上攻失败 | 偏空观察 |
| 2B失败 | 跌回后又重新收复前高 | 假突破不成立 | 中性或偏多 |

```text
顶部2B只说明冲高失败，
后续仍需观察是否演化为上涨转弱123或高位ABC。
```

## 四、ABC结构的统一定义

### 1. 术语规范

本文统一使用“点位式ABC”：

```text
A点：原趋势运行后形成的第一个关键转折点
B点：A点之后第一轮反向运动形成的关键点
C点：B点之后再次回撤或反抽形成的确认点
```

为了避免程序混淆：

- `A点、B点、C点`表示具体价格拐点；
- `A→B、B→C、C后运行`表示价格运动区间；
- 不再单独使用含义不明确的“A段、B段、C段”；
- 输出必须明确是“点”还是“区间”。

ABC是结构识别工具，不是最低点或最高点预测工具。

### 2. 低位ABC

低位ABC用于识别：

```text
下降趋势 → 止跌 → 第一次反弹 → 回踩不创新低 → 向上确认
```

| 阶段 | 必要条件 | 技术含义 | 初步等级 |
|---|---|---|---|
| A点形成 | 下跌后出现阶段低点 | 只是止跌候选，不等于底部 | C / D |
| A→B反弹 | 从A点向上形成第一轮反弹 | 观察反弹力度与压力 | C / B |
| B点形成 | 第一轮反弹遇阻形成高点 | 建立后续确认基准 | C / B |
| B→C回踩 | 从B点回落，但不应跌破A | 观察卖压是否衰减 | C / B |
| C点观察 | 暂未跌破A，但还未重新转强 | 候选确认区 | C / B |
| C点确认 | C>A，回踩缩量，并重新站回短均线 | 低位修复结构成立 | B / A |
| 突破B | 收盘有效突破B，最好有量能确认 | 低位ABC完整确认 | A / B |
| 跌破A | C或后续走势跌破A | ABC失败，恢复弱势解释 | D |

低位ABC的核心价格关系：

```text
C > A
并且价格随后有效突破B
```

标准路径：

```text
A点出现
↓
A→B反弹形成B点
↓
B→C回踩，C高于A
↓
C点企稳并重新转强
↓
有效突破B，低位ABC确认
```

### 3. 高位ABC

高位ABC用于识别：

```text
上涨趋势 → 阶段高点 → 第一轮下跌 → 反抽不创新高 → 再次向下确认
```

| 阶段 | 必要条件 | 技术含义 | 初步等级 |
|---|---|---|---|
| A点形成 | 上涨后出现阶段高点 | 只是高点，不等于顶部 | B / C |
| A→B下跌 | 从A点出现第一轮明显下跌 | 观察上涨结构是否被破坏 | C |
| B点形成 | 第一轮下跌后形成低点 | 建立后续转弱基准 | C |
| B→C反抽 | 从B点反弹，但不应突破A | 观察反抽力度 | C / D |
| C点确认 | C<A，反抽乏力，并重新转弱 | 高位转弱结构形成 | C / D |
| C后下压 | 从C点再次向B点靠近 | 转弱程度加强 | D |
| 跌破B | 收盘有效跌破B，最好有量能确认 | 高位ABC完整确认 | D |
| 收复A | 后续重新有效突破A | 高位ABC失败，结构重评 | 重评 |

高位ABC的核心价格关系：

```text
C < A
并且价格随后有效跌破B
```

标准路径：

```text
A点出现
↓
A→B下跌形成B点
↓
B→C反抽，C低于A
↓
C点再次转弱
↓
有效跌破B，高位ABC确认
```

## 五、ABC成交量验证规范

成交量不是ABC成立的绝对必要条件，但它决定结构质量和可信度。程序必须分别判断每一段的量能，而不能只输出“放量”或“缩量”一个总标签。

### 1. 低位ABC的理想量能

| 区间 | 理想量能 | 含义 |
|---|---|---|
| A点附近 | 恐慌放量后量能衰减，或出现明显承接 | 原下跌卖压可能开始衰竭 |
| A→B反弹 | 成交量逐步放大，或上涨日量能明显强于下跌日 | 有主动资金进入，不只是无量反抽 |
| B→C回踩 | 成交量逐步萎缩 | 抛压减轻，回调质量较好 |
| C点附近 | 缩量止跌，随后温和放量转强 | C点确认质量提高 |
| 突破B | 成交量明显高于近期均量 | 突破有效性提高 |

低位ABC量能正向组合：

```text
A→B放量上涨
+
B→C缩量回调
+
突破B再次放量
```

量能风险信号：

- A→B上涨无量：可能只是技术反抽；
- B→C回调持续放量：抛压仍强，C点质量下降；
- 突破B无量：假突破风险较高；
- 突破B后立即放量跌回：优先检查顶部2B或突破失败。

### 2. 高位ABC的理想量能

| 区间 | 理想量能 | 含义 |
|---|---|---|
| A点附近 | 放量滞涨、长上影或创新高但量价背离 | 上涨动能可能衰减 |
| A→B下跌 | 下跌放量或连续主动卖盘 | 资金开始退出 |
| B→C反抽 | 反弹缩量，量能弱于此前上涨段 | 反抽质量不足 |
| C点转弱 | 反抽停止，卖量重新增加 | 多头无法收复失地 |
| 跌破B | 放量跌破 | 转弱确认质量提高 |

高位ABC量能负向组合：

```text
A→B下跌有量
+
B→C反抽缩量
+
跌破B再次放量
```

风险修复信号：

- B→C反抽持续放量并接近A：高位ABC可信度下降；
- 跌破B无量并快速收回：优先检查底部2B或假跌破；
- 后续重新有效收复A：高位ABC失效。

### 3. 程序量能字段

程序必须输出：

```text
A点量能：恐慌放量 / 承接放量 / 缩量止跌 / 普通 / 数据不足
A→B量能：放量 / 温和放量 / 缩量 / 普通 / 数据不足
B→C量能：缩量 / 放量 / 普通 / 数据不足
突破确认量能：放量确认 / 无量突破 / 尚未突破 / 数据不足
量能结论：支持结构 / 部分支持 / 不支持 / 数据不足
```

不能因为缺少成交量数据而编造量能结论。数据不足时必须明确输出“数据不足”。

## 六、有效突破、确认与失效规范

### 1. 有效突破不能只看盘中触碰

程序判断“突破B”或“跌破B”时，至少检查：

- 收盘是否位于关键位之外；
- 突破幅度是否明显高于普通噪音；
- 是否连续保持，或次日未立即收回；
- 成交量是否配合；
- 是否同时站上或跌破关键均线；
- 大盘和行业是否严重反向。

输出分为：

```text
尚未突破
盘中试探
初步突破
有效突破
突破失败
```

### 2. 低位ABC的失效条件

任一情况出现，应降低或取消低位ABC解释：

- C点跌破A点；
- C点附近没有止跌迹象，并持续放量下跌；
- 突破B后快速跌回B下方且无法收复；
- 重新跌破关键前低；
- 相对强弱显著恶化。

### 3. 高位ABC的失效条件

任一情况出现，应降低或取消高位ABC解释：

- C点突破A点并形成新高；
- B→C反抽放量且重新恢复强趋势；
- 跌破B后快速收回并重新站稳；
- 后续有效收复A点；
- 相对强弱重新明显转强。

## 七、均线、动量与相对强弱验证

ABC负责结构，指标只负责验证，不能用指标替代价格结构。

### 1. 均线

| 状态 | 技术含义 |
|---|---|
| 价格站在EMA5/10/20上方，短均线向上 | 短线动量较强 |
| 跌破EMA5但守住EMA10/20 | 正常回调候选 |
| 跌破EMA20并无法修复 | 结构降级 |
| 跌破EMA50且反抽站不回 | 中期转弱 |
| 跌破EMA90 | 大级别趋势风险 |

均线周期应随K线周期变化：日线分析使用日线EMA，60分钟分析使用60分钟EMA，不能混用。

### 2. 动量

可使用MACD、RSI、KDJ或收益率斜率辅助判断，但必须避免机械打分。

低位ABC的正向动量证据包括：

- B→C过程中下跌动量弱于A点前的下跌；
- RSI或MACD出现底背离；
- C点附近动量由负转稳；
- 价格重新站回EMA5/10。

高位ABC的负向动量证据包括：

- B→C反抽动量弱于A点前上涨；
- RSI或MACD出现顶背离；
- C点附近动量由正转弱；
- 价格再次跌破EMA5/10。

### 3. 相对强弱

程序应比较标的与QQQ、SPY或行业ETF的同期表现。

```text
绝对上涨但弱于基准：结构质量需降级
绝对回调但明显强于基准：可能是健康回踩
突破时同时跑赢基准：确认质量提高
跌破时明显弱于基准：转弱质量提高
```

## 八、技术结构等级

| 等级 | 状态 | 含义 |
|---|---|---|
| A | 正向共振 | 价格结构、均线、动量、量能和相对强弱大部分一致 |
| B | 修复共振 | 已有较清晰修复迹象，但尚未完全确认 |
| C | 中性或结构未完成 | 方向不清、证据冲突或仍处于观察阶段 |
| D | 负向破位 | 结构失败、关键位破坏或弱势延续 |

等级不是交易建议，只表示技术结构质量。

## 九、ABC结构可信度

程序可输出“结构可信度”，但不得伪装成精确概率。建议使用等级或区间评分。

| 项目 | 分值 |
|---|---:|
| A、B、C点位清晰 | 20 |
| 低位C>A或高位C<A | 20 |
| 回撤/反抽量能符合结构 | 15 |
| B点突破或跌破得到确认 | 20 |
| 均线与动量支持 | 15 |
| 相对强弱支持 | 10 |

输出映射：

```text
80-100：高可信度
60-79：中高可信度
40-59：中等可信度
20-39：低可信度
0-19：不适合判断
```

如果A、B、C点本身不清晰，最高只能评为“中等可信度”；不能靠指标把不清晰的价格结构评分拉高。

## 十、当前位置标签

| 标签 | 技术含义 |
|---|---|
| A点附近 | 刚出现阶段转折点，尚未确认 |
| A点后弱弹/弱跌 | 第一轮反向运动较弱，B点未清晰 |
| B点附近 | 第一轮反向运动到达压力或支撑 |
| B→C途中 | C点尚未形成，不能提前猜测确认 |
| C点观察区 | 点位关系符合候选条件，但尚未重新转强或转弱 |
| C点确认 | 点位、量能和短线动量开始共同确认 |
| 突破B前/跌破B前 | 接近最终确认位，但尚未完成 |
| 突破B后/跌破B后 | ABC得到右侧确认 |
| 结构失败 | 跌破A、收复A或关键关系被破坏 |
| 强趋势回踩 | 趋势仍完整，不适合强行套低位ABC |
| 破位反抽 | 关键支撑已破，反抽无法站回 |
| 箱体震荡 | 在明确上下沿之间运行 |
| 结构不清 | 无法可靠识别关键前高前低 |

核心原则：

```text
B→C途中不要猜C点。
真正有价值的是C点确认，而不是主观认为“快到底”或“快到顶”。
```

## 十一、程序强制输出规范

### 1. 第一层：主判断

每次必须先输出：

```text
主判断：123 / 2B / ABC / 强趋势回踩 / 破位反抽 / 箱体震荡 / 不适合判断
方向：偏多 / 偏空 / 中性
当前阶段：具体到步骤或区间
结构演化背景：此前结构及可能演化路径
下一步确认条件：必须发生什么才升级
下一步否定条件：发生什么会降级或失效
```

### 2. 第二层：ABC字段

仅当主判断或演化背景涉及ABC时输出：

```text
ABC类型：低位ABC / 高位ABC
是否适合ABC：适合 / 暂时不清晰 / 不适合
A点：价格、时间、判定理由
B点：价格、时间、判定理由
C点：价格、时间、候选或确认状态
当前位置：A点 / A→B / B点 / B→C / C点观察 / C点确认 / 突破B前 / 突破B后 / 结构失败
点位关系：C>A / C<A / 未满足 / 数据不足
突破状态：尚未突破 / 盘中试探 / 初步突破 / 有效突破 / 突破失败
```

### 3. 第三层：验证字段

```text
均线结构：xxx
动量状态：增强 / 修复 / 衰减 / 背离 / 中性 / 数据不足
成交量分段：A点、A→B、B→C、突破确认
量能结论：支持 / 部分支持 / 不支持 / 数据不足
相对强弱：强于 / 弱于 / 接近基准 / 数据不足
关键支撑：xxx
关键压力：xxx
失效位：xxx
技术结构等级：A / B / C / D
结构可信度：高 / 中高 / 中等 / 低 / 不适合判断
```

### 4. 禁止输出

本技术策略禁止直接输出：

- 买入、卖出、加仓、减仓；
- 卖Put、卖Call或执行价建议；
- Delta、DTE、IV或权利金建议；
- 仓位和止盈方案。

可以输出：

```text
是否具备进入下一层策略分析的技术条件：是 / 否 / 等待确认
```

这只是流程状态，不是交易动作。

## 十二、结构图规范

单标的默认输出“文字说明 + 结构图”。

### 1. 图形选择

```text
主判断为ABC：使用低位或高位ABC图
主判断为2B：使用顶部或底部2B图
主判断为123：使用123三步进度图
强趋势回踩：使用趋势与均线回踩图
破位反抽：使用支撑转压力图
箱体震荡：使用箱体上下沿图
无法归类：明确写“不适合判断”
```

### 2. ABC图最低要求

- 标出A、B、C点；
- 标出当前价格所在区间；
- 标出主路径与备选路径；
- 标出确认位、降级位和失效位；
- 标出成交量是否支持结构；
- 不得画出尚未发生的C点并当作已确认事实；
- 候选点必须标注“C？”或“候选”。

### 3. 低位ABC文字图

```text
                 B（压力/确认位）
                / \
               /   \
              /     \__ C？（回踩候选）
             /             ↗
A（阶段低点）               当前价

确认：C>A，并有效突破B
失效：跌破A
```

### 4. 高位ABC文字图

```text
A（阶段高点）
 \ 
  \        C？（反抽候选）
   \      /  \
    \____/    \  当前价
         B      ↘

确认：C<A，并有效跌破B
失效：重新有效收复A
```

## 十四、常见错误

1. 把任何三段走势都叫ABC；
2. A、B点尚未明确，就提前画出C点；
3. B→C途中主观猜底或猜顶；
4. 只看价格形状，不看量能和动量；
5. 把盘中突破当成有效突破；
6. 低位ABC和高位ABC使用同一套方向语言；
7. 把2B、123和ABC重复输出成三个独立结论；
8. 把技术结构结论直接变成期权建议；
9. 使用日线均线解释小时线结构，或周期混用；
10. 数据不足时仍输出确定性结论。

## 十五、一句话原则

```text
先选择正确的结构语言，再判断当前位置。
```

```text
2B负责预警，123负责确认，ABC负责定位，量能和动量负责验证。
```

```text
低位ABC看C是否高于A并突破B；高位ABC看C是否低于A并跌破B。
```

```text
B→C途中不猜C点，只有确认后才能升级结构等级。
```

```text
技术结构只负责判断位置、方向和确认程度，不负责决定交易动作。
```
