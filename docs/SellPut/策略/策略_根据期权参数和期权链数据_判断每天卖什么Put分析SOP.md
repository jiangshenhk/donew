# Sell Put 雷达分析 SOP v1.0

> 用途：统一美股 Cash-Secured Put（现金担保 Sell Put）的筛选、比较与输出口径。  
> 目标：避免后续分析发生口径漂移，确保每次分析都能复核、比较和迭代。  
> 默认账户规模：USD 20,000。  
> 默认目标：账户长期年化 20%–30%，不是要求每一张合约都达到 20%–30%。

> **公共规则优先**：候选参数必须读取 `策略辅助_分析指标指引.md`，标的范围与禁用清单必须读取 `策略辅助_公用的股票池.md`。本文件中任何旧参数或旧示例不得覆盖公共文件。

---

## 1. 核心思想

Sell Put 不是单纯比较权利金高低，也不是只看 Delta。

统一分析框架：

```text
相近 DTE
+ 最接近 |Delta| = 0.15 的 Put
+ 实际 Bid / Ask / Mid
+ 一手现金担保金额
+ 价格安全垫
+ 盈亏平衡安全垫
+ Mid 年化
+ 技术结构质量
+ 事件风险
+ 仓位与风险因子重叠
= 是否值得卖出 Put
```

核心原则：

> **先统一概率口径，再比较收益和安全垫；最后用技术结构、事件风险和仓位集中度给安全垫打折。**

## 2. 默认扫描参数

| 项目 | 默认值 | 说明 |
|---|---:|---|
| 账户总资金 | USD 20,000 | 用于计算账户贡献年化 |
| 策略 | Cash-Secured Put | 默认按现金担保口径计算 |
| 目标 Delta | 读取公共指标指引 | 绝对值 `0.12-0.18`，优先接近 `0.15` |
| 默认期限 | 读取公共指标指引 | 新开候选统一为 4-10 DTE |
| 首选期限 | 4-10 DTE 内最接近 7 天 | 不得以更长期限替代短线候选 |
| 合约乘数 | 100 | 美股标准期权每张通常对应 100 股 |
| 权利金参考 | 同时显示 Bid 与 Mid | Bid 用于保守估算，Mid 用于横向比较 |
| 最大单手担保金额 | 读取总控账户风险规则 | 不得用本 SOP 覆盖账户桶、集中度或接货限制 |
| 技术指标周期 | 日线 | RSI、MACD、E20/E90 为主，MA200 可选 |
| 最终成交依据 | 券商实时盘口 | 截图、延迟行情和模型只能用于筛选 |

### 3.1 每个标的至少需要

| 字段 | 必须程度 | 用途 |
|---|---|---|
| 标的代码 | 必须 | 必须在公共股票池内，且不在禁用清单中 |
| 正股现价 | 必须 | 计算安全垫 |
| 到期日 | 必须 | 计算 DTE |
| Strike | 必须 | 现金担保金额与接货价 |
| Bid | 必须 | 保守收益率 |
| Ask | 必须 | 计算 Mid 与价差 |
| Delta | 必须 | 统一比较口径 |
| IV | 建议 | 判断波动率与预期波动 |
| OI | 建议 | 判断流动性 |
| Volume | 建议 | 判断当日活跃度 |
| 财报 / 重大事件日期 | 建议 | 尾部风险过滤 |

### 3.2 截图要求

每个标的优先截取：

```text
相同到期日
+ Put
+ Delta 位于 0.12–0.18 的 Strike；如需要展示相邻档位，必须标注“仅作邻档比较”
+ Bid / Ask / Delta / IV / OI / Volume
```

如某档 Delta 被遮挡或截图缺失：

- 允许使用相邻 Strike 临时估算；
- 必须标注“截图遮挡 / Delta 待确认”；
- 不得把估算值伪装成真实盘口。

## 4. 标准计算公式

### 4.1 Mid

```text
Mid = (Bid + Ask) ÷ 2
```

### 4.2 一手现金担保金额

```text
现金担保金额 = Strike × 100
```

### 4.3 价格安全垫

```text
价格安全垫 = (现价 - Strike) ÷ 现价
```

含义：股价还能下跌多少，才触及 Strike。

### 4.4 盈亏平衡价

```text
盈亏平衡价 = Strike - Mid
```

### 4.5 盈亏平衡安全垫

```text
盈亏平衡安全垫
= [现价 - (Strike - Mid)] ÷ 现价
```

含义：考虑收到权利金后，股价还能下跌多少，才进入亏损区。

### 4.6 单周期保守收益率

```text
单周期保守收益率 = Bid ÷ Strike
```

### 4.7 单周期预计收益率

```text
单周期预计收益率 = Mid ÷ Strike
```

### 4.8 统一年化

```text
Mid = (Bid + Ask) ÷ 2
年化 = Mid ÷ Strike × 365 ÷ DTE
```

用途：候选准入和横向比较的唯一年化口径。注意：不代表全年能够机械复制。

### 4.9 对账户贡献年化

```text
账户贡献年化
= Mid × 100 ÷ 账户总资金 × 365 ÷ DTE
```

用途：判断卖出一张 Put 后，对整个账户目标收益的实际贡献。

## 5. 统一比较规则

### 5.1 必须先统一 DTE

不能把 7 DTE 与 20 DTE 的 Put 直接混排。

正确做法：

- 7 DTE 候选放一张表；
- 14 DTE 候选放另一张表；
- QLD 等只有更长期限的合约，单独列示；
- 可以比较，但必须明确不同期限。

### 5.2 必须在公共 Delta 范围内选择最接近 `|Delta| = 0.15` 的 Put

统一 Delta 后，才能公平比较：

- 收益率；
- Strike 安全垫；
- 盈亏平衡安全垫；
- 一手金额；
- 技术结构质量。

注意：

> Delta 接近 0.15 不等于真实被行权概率一定是 15%。  
> Delta 只是统一筛选口径，不是风险保证。

### 5.3 必须显示 Bid、Ask、Mid，但年化只按 Mid

| 指标 | 用途 |
|---|---|
| Bid / Ask | 用于核验盘口、价差与限价空间 |
| Mid | `(Bid + Ask) / 2`，唯一年化计算口径 |
| 实际成交价 | 用于交易复盘，不改变候选年化比较口径 |

### 5.4 不得用模型权利金替代真实盘口

模型可用于：

- 预筛选；
- 估算预计下沿；
- 数据缺失时提供参考。

模型不得伪装成：

- 实际 Bid；
- 实际 Mid；
- 可成交价格。

## 6. 技术安全垫评估

账面安全垫不是最终安全垫。

需要结合：

```text
MACD
+ RSI
+ 股价相对 E20 / E90 的位置
+ 可选 MA200
+ 当前处于上涨、回调、下跌还是过热阶段
```

### 6.1 技术结构分类

| 结构 | 特征 | 处理方式 |
|---|---|---|
| 健康上涨 | MACD 为正；RSI 45–70；股价在 E20、E90 上方 | 安全垫质量较高 |
| 上涨中回调 | 股价回踩 E20 或 E90；MACD 尚未明显恶化；RSI 回落 | 可重点观察 |
| 下跌中 | MACD 为负；跌破 E20、E90；RSI 偏弱 | 安全垫需要打折，避免接第一根飞刀 |
| 短线过热 | RSI > 75；远离 E20 / E90；快速拉升 | 等待回踩或横盘消化；强势股按公共指标指引的技术优先分支处理，不自行降低 Delta 到公共范围外 |
| 结构破坏 | 跌破关键支撑、E90 或长期均线后反弹无力 | 暂停 Sell Put 或显著降低 Strike |

### 6.2 技术安全垫质量分级

| 等级 | 含义 | 常见特征 |
|---|---|---|
| A | 安全垫质量高 | MACD 正向；RSI 不过热；盈亏平衡价低于重要均线 |
| B | 可做但需控制仓位 | 有回调或杠杆 ETF 风险；仍有结构支撑 |
| C | 账面安全垫存在，但技术质量偏弱 | MACD 负向、RSI 过热或弱势反弹 |
| D | 暂不适合 | 结构破坏、事件风险突出或流动性明显不足 |

### 6.3 技术指标解释规则

不要机械使用指标：

- RSI 高，不代表立刻下跌，但代表追高风险增加；
- RSI 低，不代表一定见底，可能仍处于下跌趋势；
- MACD 为正，不代表无风险；
- MACD 为负，不代表完全不能做，但需要更低 Strike；
- 均线位置用于判断“安全垫落在哪里”，不是单独预测价格。

### 6.4 高风险标的的额外折扣

| 标的类型 | 额外规则 |
|---|---|
| 高波动单股 | 最多一张；优先要求盈亏平衡价落在重要支撑下方 |
| 2 倍 ETF | 目标 Strike 可比普通 ETF 再低一档 |
| 3 倍 ETF | 只做战术仓；避免与同指数杠杆 ETF 重复暴露 |
| BTC / Crypto 相关 | 必须检查 BTC 总控开关与相关仓位重叠 |
| 财报窗口 | 原则上避开，除非明确接受跳空风险 |

## 7. 风险因子与仓位集中度

不能只看“还有多少现金”，还要看风险因子是否重复。

### 7.1 风险因子示例

| 风险因子 | 标的示例 |
|---|---|
| Nasdaq 科技风险 | QLD、TQQQ |
| BTC / Crypto 风险 | IBIT、MSTR、MARA、RIOT、HOOD |
| 单一个股风险 | INTC、HOOD、SMCI、PLTR |
| 黄金矿业风险 | GDX |

### 7.2 仓位规则

| 规则 | 默认要求 |
|---|---|
| 单一高波动个股 | 同时最多一张 |
| MSTR | 一张可能占用大部分资金，只做特殊窗口 |
| QLD + TQQQ | 不应同时重仓 |
| MSTR + IBIT + MARA / RIOT + HOOD | 不能误认为是充分分散 |
| 保留现金 | 常态保留约 20%–40%，用于滚仓与机会仓 |

| 排名 | 标的与候选 Put | Mid | Mid 年化 | 盈亏平衡价 | 账面安全垫 | MACD | RSI | E20 / E90 位置 | 技术安全垫质量 | 结论 |
|---:|---|---:|---:|---:|---:|---|---:|---|---|---|

### 输出物 3：账户适配度表

| 标的 | 一手担保金额 | 占账户比例 | 一周期权利金 | 对账户贡献年化 | 风险因子 | 是否与现有仓位重叠 |
|---|---:|---:|---:|---:|---|---|

用途：防止单张收益率很好，但账户结构失衡。

---

### 输出物 4：最终优先级

分为三档：

| 分组 | 含义 |
|---|---|
| 第一梯队：优先确认 | 收益、安全垫、技术结构和仓位颗粒度较均衡 |
| 第二梯队：等待确认 | 数据不错，但有趋势、事件或集中度问题 |
| 第三梯队：暂不急 | 过热、弱势、价差宽、流动性差或收益不足 |

每个标的必须给出一句原因。

---

### 输出物 5：异常与数据质量说明

每次必须明确标注：

- 是否为真实券商截图；
- 是否存在延迟；
- 是否使用模型估算；
- 哪些 Delta 被遮挡；
- 哪些到期日不同；
- 哪些标的存在财报或重大事件；
- 哪些技术指标来源不完整；
- 是否需要回券商确认实时 Bid / Ask。

## 9. 最终决策逻辑

### 9.1 基础公式

```text
候选资格
=
4-10 DTE
+ |Delta| 位于 0.12-0.18，优先接近 0.15
+ Mid 年化 >= 30%
+ 流动性可接受
+ 一手金额可承接
```

### 9.2 综合决策

```text
是否值得卖 Put
=
Mid 年化
+ 盈亏平衡安全垫
+ 技术安全垫质量
+ 一手金额
+ 流动性
+ 事件风险
+ 风险因子重叠
```

### 9.3 不允许只用单一指标决策

错误方式：

```text
只看 Mid 年化最高
```

错误方式：

```text
只看安全垫最大
```

错误方式：

```text
只看 Delta 约 0.15
```

正确方式：

```text
先统一 Delta
→ 再比较年化与安全垫
→ 再评估技术结构
→ 再检查账户集中度
→ 最后回券商确认实时盘口
```
