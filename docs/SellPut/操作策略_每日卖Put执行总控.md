# 操作策略_每日卖Put执行总控

> 本文件是 `docs/SellPut/` 目录下的每日 Sell Put 操作总控入口。  
> 它不替代任何单项策略文件，而是规定每天应该如何调用“市场风向判断”“动量与 ABC 技术结构判断”“期权链参数判断”三个模块，最终形成可执行的卖 Put 决策。

---

## 一、总控目标

本策略的目标不是寻找最高权利金，而是寻找：

```text
市场允许 + 技术结构健康 + 期权参数合理 + 自己愿意接货
```

只有这四个条件同时满足，才是高质量 Sell Put 机会。

每日操作必须遵守一个核心顺序：

```text
先判断市场能不能做
↓
再判断哪个标的能做
↓
最后才判断卖哪一个 Strike
```

严禁因为某个 Put 权利金突然变高，就绕过市场风向和技术结构判断直接交易。

---

## 二、每日操作总流程

每日 Sell Put 执行顺序如下：

```text
Step 1：确定今日市场风向
Step 2：确定股票池中各标的的技术结构
Step 3：筛选候选标的
Step 4：读取期权链截图或期权链参数
Step 5：横向比较候选 Put
Step 6：输出最终交易结论
Step 7：记录失效条件和复盘要点
```

其中前三步决定“能不能做、做谁”，第四至第六步决定“卖哪一档、卖几张、什么条件下取消”。

---

## 三、股票池与扫描权限

本策略只扫描固定股票池，不因为某个标的当日权利金突然变高，就临时加入交易。

股票池分为三类：

1. 核心执行池：可以真正进入 Sell Put 候选。
2. 风向观察池：主要用于判断市场环境，不直接作为交易对象。
3. 暂停/候补池：暂时不主动交易，只有重新达标后才恢复扫描。

---

### 3.1 核心执行池

核心执行池是每日 Sell Put 的主要候选范围。

| 标的 | 类型 | 主要风险因子 | 默认扫描权限 | 备注 |
|---|---|---|---|---|
| QLD | 纳指 2 倍 ETF | Nasdaq / 科技股 / 杠杆 ETF | 常规扫描 | 主力 Sell Put 标的，适合作为账户核心轮动标的 |
| MSTR | BTC 高波动代理 | BTC / 公司溢价 / 融资风险 | 仅顺风小仓 | 必须结合 BTC、IBIT、Crypto 情绪判断 |
| INTC | 半导体个股 | 芯片周期 / 公司事件 / 政策 | 事件过滤后扫描 | 更适合在结构修复和事件落地后做 |
| CRCL | Crypto / Fintech 个股 | 加密风险 / 监管 / 高波动 | 谨慎扫描 | 波动大，不能只看权利金，需要严格控制 Delta 和仓位 |

---

### 3.2 风向观察池

风向观察池不一定直接卖 Put，主要用于辅助判断核心执行池是否处于顺风环境。

| 观察对象 | 用途 | 对应影响 |
|---|---|---|
| QQQ | 判断纳指和大型科技股风险偏好 | 影响 QLD |
| SPY | 判断美股整体风险偏好 | 影响所有标的 |
| IWM | 判断市场宽度和小盘股风险偏好 | 辅助判断风险承接能力 |
| SMH / SOXX | 判断半导体和 AI 主线 | 影响 INTC、QLD |
| BTC / IBIT | 判断 Crypto 风险偏好 | 影响 MSTR、CRCL |
| VIX | 判断波动率风险 | 影响全部 Sell Put 环境 |
| 10Y 美债收益率 | 判断利率压力 | 影响成长股、科技股、杠杆 ETF |
| DXY 美元指数 | 判断美元流动性压力 | 影响风险资产 |
| WTI 原油 | 判断通胀和地缘风险 | 影响宏观风险偏好 |

规则：

```text
每日操作只从“核心执行池”里选标的。
不能从“风向观察池”直接跳到交易。
```

---

### 3.3 暂停/候补池

如果某个标的出现以下情况，自动进入暂停池：

- 财报或重大事件临近；
- 股价放量跌破关键支撑；
- ABC 结构进入明显下跌段；
- 跌破 50 日线后没有修复；
- Put 价差过大、OI 过低、流动性不足；
- 与现有仓位风险因子高度重叠；
- 市场风向为逆风或事件风；
- 用户主观上并不愿意接货。

暂停池标的只有在市场风向、技术结构和期权链条件重新达标后，才可以恢复扫描。

## 四、每日必须参考的三个策略文件

本总控文件需要调用以下三个子策略文件。

---

### 4.1 市场风向判断文件

参考文件：

```text
策略_待卖put的股票池里面各个票的市场风向研究模式.md
```

作用：

判断未来 1–7 个交易日，当前市场环境是否允许在股票池中卖 Put。

重点判断：

- 大盘方向；
- QQQ / SPY / IWM 风险偏好；
- VIX 是否异常升高；
- 美债收益率是否冲击成长股估值；
- 美元指数是否带来流动性压力；
- BTC 是否影响 MSTR / CRCL；
- AI、半导体、科技股主线是否继续；
- 个股是否存在新闻、监管、财报、诉讼、政策等事件。

输出结果：

```text
市场风向等级：A / B / C / D
```

| 等级 | 风向 | 含义 | 操作 |
|---|---|---|---|
| A | 顺风 | 市场支持卖 Put | 可以进入技术结构判断 |
| B | 横风 | 市场没有明显风险，但也不是强顺风 | 小仓、远虚值、保守执行 |
| C | 逆风 | 市场风险上升，安全垫容易失效 | 原则上暂停新开仓 |
| D | 事件风 | 重大事件未落地，风险不可控 | 不主动卖 Put |

---

### 4.2 动量与 ABC 技术结构判断文件

参考文件：

```text
策略_待卖put的股票池里面各个票根据动量原理和abc结构判断个股的策略.md
```

作用：

判断核心执行池中的每个标的，当前是否处于适合卖 Put 的技术结构。

重点判断：

- 当前价格相对 5 日、20 日、50 日、90 日均线的位置；
- 是否处于 A/B/C 结构中的健康阶段；
- 是否缩量回调；
- 是否放量下跌；
- 是否跌破关键支撑；
- 是否出现修复信号；
- 是否强于大盘；
- 是否属于假反弹或诱多结构。

输出结果：

```text
技术结构等级：A / B / C / D
```

| 等级 | 技术结构 | 含义 | 操作 |
|---|---|---|---|
| A | 正向共振 | 趋势健康，回调可接 | 优先候选 |
| B | 修复共振 | 有修复迹象，但未完全确认 | 小仓或等待更好位置 |
| C | 中性震荡 | 方向不清楚 | 只观察或卖更远虚值 |
| D | 负向破位 | 技术结构破坏 | 不卖 Put |

---

### 4.3 期权链参数判断文件

参考文件：

```text
策略_根据期权参数和期权链数据_判断每天卖什么Put分析SOP.md
```

作用：

在市场风向和技术结构都允许的情况下，根据期权链截图或期权链参数，判断具体卖哪一个 Put。

重点判断：

- DTE；
- Strike；
- Delta；
- Bid / Ask / Mid；
- IV；
- Volume；
- Open Interest；
- 安全垫；
- 盈亏平衡价；
- 年化收益率；
- 账户资金占用；
- 是否临近财报或重大事件；
- 是否与已有仓位风险因子重叠。

输出结果：

```text
期权链等级：A / B / C / D
```

| 等级 | 期权链状态 | 含义 | 操作 |
|---|---|---|---|
| A | 权利金合理、流动性好、安全垫足够 | 可以执行 | 正常考虑 |
| B | 基本可做，但有小瑕疵 | 谨慎执行 | 小仓或压低 Strike |
| C | 性价比一般 | 不急 | 等更好价格 |
| D | 流动性差或风险过高 | 不交易 | 放弃 |

## 五、评分与最终决策规则

最终操作等级取三关中的最低等级：

```text
最终等级 = 市场风向等级、技术结构等级、期权链等级中的最低等级
```

### 5.1 决策示例

| 市场风向 | 技术结构 | 期权链 | 最终操作 |
|---|---|---|---|
| A | A | A | 可以卖 Put |
| A | B | A | 可以小仓卖 Put |
| A | C | A | 暂缓，等技术确认 |
| B | A | A | 小仓、远虚值 |
| C | A | A | 不卖 |
| D | A | A | 不卖，等事件落地 |
| A | A | C | 不急，等更好价格 |
| A | A | D | 不交易 |
| B | B | B | 可小仓试单，但必须降低 Delta |
| A | D | A | 不卖，技术破位不能用权利金补偿 |

## 六、Delta 使用规则

期权链分析中，Delta 分为两种口径：

```text
比较 Delta：用于横向比较不同标的的期权性价比。
执行 Delta：用于真实下单，需要根据标的风险和市场风向调整。
```

### 6.1 比较 Delta

默认使用接近：

```text
|Delta| ≈ 0.15
```

用于横向比较：

- 哪个标的权利金更合理；
- 哪个标的安全垫更充足；
- 哪个标的年化收益更高；
- 哪个标的风险补偿更合理。

### 6.2 执行 Delta

真实下单时，不同标的不能机械使用同一个 Delta。

| 标的 | 顺风环境 | 横风环境 | 逆风/事件风 |
|---|---|---|---|
| QLD | 0.10–0.20 | 0.08–0.12 | 不开新仓 |
| MSTR | 0.05–0.10 | 0.03–0.06 | 不开新仓 |
| INTC | 0.10–0.15 | 0.08–0.10 | 不开新仓 |
| CRCL | 0.05–0.10 | 0.03–0.06 | 不开新仓 |

原则：

```text
波动越大，Delta 越低。
事件越近，Delta 越低。
市场越逆风，越不应该新开仓。
```

## 七、DTE 使用规则

默认优先观察：

```text
7–14 DTE：短线周度 Sell Put
21–35 DTE：月度主线 Sell Put
```

### 7.1 周度 Sell Put

适合条件：

- 市场风向 A 或 B；
- 技术结构 A 或 B；
- 标的流动性好；
- 期权价差合理；
- 没有重大事件。

特点：

- 收益快；
- 容易滚动；
- 但对短期波动更敏感。

### 7.2 月度 Sell Put

适合条件：

- 有较大安全垫；
- 权利金明显比周度更合理；
- 用户愿意接货；
- 账户资金占用可以接受。

特点：

- 年化更容易稳定；
- 时间缓冲更大；
- 但持仓周期更长。

## 八、仓位与风险因子规则

### 8.1 单标的仓位

单一标的不能因为权利金高而过度集中。

建议：

| 标的 | 单次新开仓建议 |
|---|---|
| QLD | 可作为主力，但需要控制总张数 |
| MSTR | 小仓，不能和 BTC / COIN / CRCL 风险过度叠加 |
| INTC | 事件过滤后小到中等仓位 |
| CRCL | 小仓，只在顺风且流动性合格时考虑 |

---

### 8.2 风险因子重叠

以下标的或资产具有较高风险相关性，不能同时重仓：

| 风险因子 | 相关标的 |
|---|---|
| Nasdaq / 科技股 | QLD、QQQ、NVDA、MSFT、AAPL、AMZN、META、GOOG |
| AI / 半导体 | QLD、INTC、SMH、SOXX、NVDA、AMD、AVGO |
| BTC / Crypto | MSTR、CRCL、COIN、IBIT、BTC |
| 利率敏感资产 | QLD、成长股、科技股、长久期资产 |

如果已有仓位已经暴露在某个风险因子上，新开仓必须降低张数、降低 Delta 或放弃交易。

## 九、一票否决规则

只要出现以下任意情况，即使权利金很高，也不主动卖 Put：

1. 市场风向为 C 或 D；
2. VIX 快速上升，市场进入风险释放期；
3. 个股放量跌破关键支撑；
4. 个股跌破 ABC 结构中的关键低点；
5. 股价跌破 50 日线后没有修复；
6. 财报、监管、诉讼、政策等重大事件未落地；
7. Put 权利金突然变肥，但同时股价破位；
8. Bid / Ask 价差过大；
9. Open Interest 太低；
10. 账户资金占用超过计划上限；
11. 同一风险因子已经暴露过高；
12. 用户主观上不愿意按 Strike 接货；
13. 只能靠“我觉得应该会反弹”来解释交易；
14. 年化收益很高，但安全垫不足；
15. 标的进入暂停池。

一句话：

```text
权利金不能补偿结构性破位。
```

## 十二、交易纪律

1. 不因为权利金高而卖 Put。
2. 不因为年化收益率高而忽略安全垫。
3. 不在市场逆风时为了交易而交易。
4. 不在技术破位时用“应该会反弹”安慰自己。
5. 不在重大事件前重仓卖 Put。
6. 不在流动性差的期权链里勉强成交。
7. 不让单一风险因子占用过多账户资金。
8. 不卖自己不愿意接货的 Strike。
9. 不把观察池标的临时变成交易标的。
10. 不让期权链反过来决定股票池。

## 十四、最终原则

本策略的核心不是每天都要交易，而是等到赔率合适时出手。

```text
市场顺风时，卖 Put 是收租。
市场逆风时，卖 Put 是接刀。
```

每天最终只需要回答一个问题：

```text
今天这个 Put，是我愿意接货的好价格，还是市场用高权利金诱惑我去承担不该承担的风险？
```

如果答案不清楚，就不交易。
