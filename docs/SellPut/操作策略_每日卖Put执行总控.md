# 操作策略_每日卖Put执行总控

> 本文件是 `docs/SellPut/` 目录下的每日 Sell Put 操作总控入口。  
> 它不替代任何单项策略文件，而是规定每天应该如何调用“市场风向判断”“动量与 ABC 技术结构判断”“期权链参数判断”三个模块，最终形成可执行的卖 Put 决策。

---

## 一、总控目标

本策略的目标不是寻找最高权利金，而是寻找：

```text
市场允许 + 技术结构健康 + 期权参数合理 + 自己愿意接货
```

只有这四个条件同时满足，才是高质量 Sell Put 机会。

每日操作必须遵守一个核心顺序：

```text
先判断市场能不能做
↓
再判断哪个标的能做
↓
最后才判断卖哪一个 Strike
```

严禁因为某个 Put 权利金突然变高，就绕过市场风向和技术结构判断直接交易。

---

## 二、每日操作总流程

每日 Sell Put 执行顺序如下：

```text
Step 1：确定今日市场风向
Step 2：确定股票池中各标的的技术结构
Step 3：筛选候选标的
Step 4：读取期权链截图或期权链参数
Step 5：横向比较候选 Put
Step 6：输出最终交易结论
Step 7：记录失效条件和复盘要点
```

其中前三步决定“能不能做、做谁”，第四至第六步决定“卖哪一档、卖几张、什么条件下取消”。

---

## 三、股票池与扫描权限

本策略只扫描固定股票池，不因为某个标的当日权利金突然变高，就临时加入交易。

股票池分为五类：

1. 核心执行池：可以真正进入 Sell Put 候选。
2. ETF 扩展扫描池：优先寻找指数、行业、主题 ETF 中 7 天左右、Delta 0.15 附近、年化 40% 以上的机会。
3. 热门股扩展扫描池：只在接货金额、技术结构、波动率补偿都合格时进入候选。
4. 风向观察池：主要用于判断市场环境，不直接作为交易对象。
5. 暂停/禁用池：暂时不主动交易或明确禁止交易，只有重新达标或用户解除禁用后才恢复扫描。

### 3.1 核心执行池

核心执行池是每日 Sell Put 的主要候选范围。

| 标的 | 类型 | 主要风险因子 | 默认扫描权限 | 备注 |
|---|---|---|---|---|
| QLD | 纳指 2 倍 ETF | Nasdaq / 科技股 / 杠杆 ETF | 常规扫描 | 主力 Sell Put 标的，适合作为账户核心轮动标的 |
| EEM | 新兴市场 ETF | 美元 / 美债利率 / 中国与亚洲资产 / 全球风险偏好 | 常规扫描 | 稳健 ETF 候选，适合用高 IV Percentile 换安全垫，不追近价 |
| MSTR | BTC 高波动代理 | BTC / 公司溢价 / 融资风险 | 仅顺风小仓 | 必须结合 BTC、IBIT、Crypto 情绪判断 |
| INTC | 半导体个股 | 芯片周期 / 公司事件 / 政策 | 事件过滤后扫描 | 更适合在结构修复和事件落地后做 |

用户账户当前明确禁用：

```text
CRCL 禁用，不进入实盘扫描和下单候选。
```

### 3.1.1 EEM 纳入核心执行池的定位

EEM 是新兴市场 ETF，不是高弹性主攻标的，而是“稳健 ETF 收租候选”。

适合 EEM 的情景：

```text
美元不强势上行 + 美债利率不快速冲高 + 全球风险偏好稳定 + EEM 技术结构未破位 + Barchart IV Percentile 偏高
```

EEM 的核心用法不是追求最高权利金，而是：

```text
用相对偏高的 IV Percentile，换取更远 Strike 和更大安全垫。
```

EEM 不适合在以下情况主动卖 Put：

- DXY 快速上行；
- 10Y 美债收益率快速冲高；
- 中国 / 亚洲市场出现明显风险事件；
- 全球风险偏好转弱，SPY、QQQ、IWM 同时走弱；
- EEM 放量跌破关键支撑；
- 权利金不肥但价格已经明显涨高。

---

### 3.2 ETF 扩展扫描池

ETF 扩展扫描池用于寻找分散度更高、单票暴雷风险更低、但权利金仍然相对肥的 Sell Put 机会。

ETF 初筛硬条件：

```text
DTE：5–10 天，核心看 7 天左右
Delta：0.12–0.18，核心看 0.15 附近
年化收益：>= 40%
Strike 安全垫：最好 >= 7%
技术结构：不能处在明确下降通道，不能在双头位置盲目卖
成交方式：价差不是硬门槛，但必须限价，不追 Ask
```

| 标的 | 类型 | 默认扫描权限 | 备注 |
|---|---|---|---|
| EEM | 新兴市场 ETF | 常规扫描 | 账户友好，但若 K 线双头或年化不足则放弃 |
| QLD | 纳指 2 倍 ETF | 常规扫描 | 可与核心池重合，优先看 5–10 DTE |
| TQQQ | 纳指 3 倍 ETF | 小仓扫描 | 权利金更肥，但只允许极小仓 |
| GDX | 金矿股 ETF | 条件扫描 | 下降通道中不卖，等破前低后止跌再看 |
| ARKK | 高 beta 成长主题 ETF | 条件扫描 | 只在 IV 足够肥且技术不破位时看 |
| IBIT | BTC ETF | 条件扫描 | 不能与 MSTR / MARA / RIOT 风险叠加过高 |

ETF 风向参考但不优先直接交易：

```text
QQQ, SPY, IWM, SMH, SOXX, XLK, XLE, XLF, SLV, GLD
```

其中 `QQQ / SPY / SMH / SOXX` 主要用于判断市场和板块风向；如果单张接货金额过大，不直接作为 2 万美元账户的 Cash Secured Put 主标的。

### 3.3 热门股扩展扫描池

热门股扩展扫描池只用于寻找账户金额可承受、权利金明显高于 ETF、且技术结构没有失控的机会。热门股不是每日必做池。

热门股初筛条件：

```text
DTE：5–10 天
Delta：0.12–0.18
年化收益：>= 40%
Strike 安全垫：最好 >= 7%
单张接货金额：原则上不超过账户 50%
已有同类风险因子时必须降级或放弃
```

| 标的 | 类型 | 默认扫描权限 | 备注 |
|---|---|---|---|
| SOFI | 金融科技 / 消费金融 | 常规扫描 | 小账户友好，适合练习策略 |
| RIVN | 电动车个股 | 常规扫描 | 小账户友好，但需确认技术止跌 |
| SMCI | AI 服务器 / 高波动硬件 | 条件扫描 | 年化可能合格，但个股风险较高 |
| INTC | 半导体个股 | 条件扫描 | 与核心池重合，需事件过滤 |
| MSTR | BTC 高波动代理 | 仅顺风小仓 | 已有仓位时不叠加 |
| MARA | BTC 矿股 | 实验小仓 | 高波动，不作为核心 |
| RIOT | BTC 矿股 | 实验小仓 | 高波动，不作为核心 |
| IONQ | 量子计算高波动股 | 实验小仓 | 真波动很高，需更大安全垫 |
| HOOD | 券商 / 交易平台 | 条件扫描 | 接货金额偏大，需谨慎 |
| PLTR | 高估值软件 / AI 概念 | 观察扫描 | IV Rank 低时不优先 |

大票和高价热门股主要作为风向参考，不作为 2 万美元账户的优先 CSP 标的：

```text
NVDA, MSFT, AAPL, AMZN, META, GOOGL, AVGO, AMD, TSLA
```

### 3.4 风向观察池

风向观察池不一定直接卖 Put，主要用于辅助判断核心执行池是否处于顺风环境。

| 观察对象 | 用途 | 对应影响 |
|---|---|---|
| QQQ | 判断纳指和大型科技股风险偏好 | 影响 QLD |
| SPY | 判断美股整体风险偏好 | 影响所有标的 |
| IWM | 判断市场宽度和小盘股风险偏好 | 辅助判断风险承接能力 |
| SMH / SOXX | 判断半导体和 AI 主线 | 影响 INTC、QLD |
| EEM / FXI / KWEB | 判断新兴市场、中国资产和亚洲风险偏好 | 影响 EEM |
| BTC / IBIT | 判断 Crypto 风险偏好 | 影响 MSTR、MARA、RIOT、IBIT |
| VIX | 判断波动率风险 | 影响全部 Sell Put 环境 |
| 10Y 美债收益率 | 判断利率压力 | 影响成长股、科技股、杠杆 ETF、新兴市场 |
| DXY 美元指数 | 判断美元流动性压力 | 影响风险资产与 EEM |
| WTI 原油 | 判断通胀和地缘风险 | 影响宏观风险偏好 |

规则：

```text
每日操作优先从“核心执行池”和“ETF 扩展扫描池”里选标的。
热门股扩展扫描池只在权利金、安全垫、技术结构、账户占用都合格时进入候选。
不能从“风向观察池”直接跳到交易。
```

### 3.5 暂停/禁用池

如果某个标的出现以下情况，自动进入暂停池：

- 财报或重大事件临近；
- 股价放量跌破关键支撑；
- ABC 结构进入明显下跌段；
- 跌破 50 日线后没有修复；
- Put 价差过大、OI 过低、流动性不足；
- 与现有仓位风险因子高度重叠；
- 市场风向为逆风或事件风；
- 用户主观上并不愿意接货。

明确禁用标的：

| 标的 | 状态 | 原因 |
|---|---|---|
| CRCL | 禁用 | 用户明确要求不能做 |

暂停池标的只有在市场风向、技术结构和期权链条件重新达标后，才可以恢复扫描。禁用池标的只有用户明确解除禁用后才可以恢复扫描。

SOFI, RIVN, SMCI, INTC, MSTR, MARA, RIOT, IONQ, HOOD, PLTR

## 四、每日必须参考的三个策略文件

本总控文件需要调用以下三个子策略文件。

### 4.1 市场风向判断文件

参考文件：

```text
策略_待卖put的股票池里面各个票的市场风向研究模式.md
```

作用：判断未来 1–7 个交易日，当前市场环境是否允许在股票池中卖 Put。

重点判断：

- 大盘方向；
- QQQ / SPY / IWM 风险偏好；
- VIX 是否异常升高；
- 美债收益率是否冲击成长股估值；
- 美元指数是否带来流动性压力；
- EEM / FXI / KWEB 是否影响新兴市场 ETF 方向；
- 日元加息、日元快速升值、日元 carry trade 平仓是否带来全球流动性冲击；
- BTC 是否影响 MSTR / MARA / RIOT / IBIT；
- AI、半导体、科技股主线是否继续；
- 个股是否存在新闻、监管、财报、诉讼、政策等事件。

输出结果：

```text
市场风向等级：A / B / C / D
```

| 等级 | 风向 | 含义 | 操作 |
|---|---|---|---|
| A | 顺风 | 市场支持卖 Put | 可以进入技术结构判断 |
| B | 横风 | 市场没有明显风险，但也不是强顺风 | 小仓、远虚值、保守执行 |
| C | 逆风 | 市场风险上升，安全垫容易失效 | 原则上暂停新开仓 |
| D | 事件风 | 重大事件未落地，风险不可控 | 不主动卖 Put |

#### 4.1.1 日元 Carry Trade 风险专项检查

每日市场风向判断中，必须单独检查“日元 carry trade 平仓风险”，并在日报中单独输出提醒。

该风险的核心逻辑是：

```text
日本加息或日元快速升值
→ 借日元买美元资产的资金需要平仓
→ 卖出流动性最好的美元资产
→ QQQ / 纳指 / 七姐妹 / AI / 半导体先承压
→ VIX 和 Put 权利金上升，但这可能是流动性踩踏，不是可卖的错价
```

每日必须检查以下指标：

| 指标 | 观察重点 | 风险含义 |
|---|---|---|
| USDJPY | 是否快速下跌，日元是否突然升值 | Carry trade 平仓压力上升 |
| 日本 10Y 国债收益率 / JGB | 是否快速上行 | 日本资金成本上升 |
| 日本央行 / 日本财务省表态 | 是否出现加息、干预汇率、密切关注汇率等表述 | 政策触发风险 |
| VIX | 是否快速上升 | 风险资产波动放大 |
| QQQ vs SPY | QQQ 是否明显弱于 SPY | 流动性资产被优先抛售 |
| SMH / SOXX / NVDA / 七姐妹 | 是否领跌 | AI/半导体拥挤交易被平仓 |
| DXY / 美债收益率 | 是否异常波动 | 美元流动性和利率冲击 |

专项输出必须使用以下三档：

```text
日元 Carry 风险：无明显苗头 / 有苗头 / 已触发降级
```

降级规则：

- 如果 USDJPY 快速下破关键位，或日元单日/两日内急升：纳指类 Sell Put 自动降级；
- 如果 QQQ 明显弱于 SPY，且 SMH / SOXX / 七姐妹同步走弱：QLD / TQQQ / SOXL 不主动开新仓；
- 如果 VIX 同步快速上升：所有高 beta Put 降级，只允许观察；
- 如果日元 carry 风险已经触发，不因 Put 年化达到 40% 以上而交易；
- 已有仓位不加仓，只允许减风险、等止跌、等 IV 稳定后再评估。

执行原则：

```text
第一天不接刀；
第二天看是否继续抛售；
只有出现止跌、缩量、IV 仍高，才重新考虑远虚值 Put。
```

### 4.2 动量与 ABC 技术结构判断文件

参考文件：

```text
策略_待卖put的股票池里面各个票根据动量原理和abc结构判断个股的策略.md
```

作用：判断核心执行池中的每个标的，当前是否处于适合卖 Put 的技术结构。

重点判断：

- 当前价格相对 5 日、20 日、50 日、90 日均线的位置；
- 是否处于 A/B/C 结构中的健康阶段；
- 是否缩量回调；
- 是否放量下跌；
- 是否跌破关键支撑；
- 是否出现修复信号；
- 是否强于大盘；
- 是否属于假反弹或诱多结构。

输出结果：

```text
技术结构等级：A / B / C / D
```

| 等级 | 技术结构 | 含义 | 操作 |
|---|---|---|---|
| A | 正向共振 | 趋势健康，回调可接 | 优先候选 |
| B | 修复共振 | 有修复迹象，但未完全确认 | 小仓或等待更好位置 |
| C | 中性震荡 | 方向不清楚 | 只观察或卖更远虚值 |
| D | 负向破位 | 技术结构破坏 | 不卖 Put |

字段约定：

| 字段 | 含义 |
|---|---|
| `trend_bias` | `bullish / neutral / bearish` |
| `position_state` | `near_prior_high / near_prior_low / inside_box / reclaiming / breaking_down` |
| `structure_type` | `breakout_pullback / breakdown_retest / bottom_2b / top_2b / low_abc / high_abc / weak_repair / box_range` |
| `structure_confirmation` | `confirmed / unconfirmed / watching / failed` |
| `prior_high` | 当前结构最重要前高 |
| `prior_low` | 当前结构最重要前低 |
| `support_zone` | 支撑或前低观察区 |
| `resistance_zone_1` | 第一压力区 |
| `resistance_zone_2` | 第二压力区 |
| `ema_state` | 当前价格相对 EMA 的约束关系 |
| `macd_state` | 动量方向 |
| `rsi_state` | 强弱区状态 |
| `volume_state` | 放量、缩量、普通量状态 |
| `action` | `can_sell_put / cautious / observe / pause` |
| `summary` | 一句话技术结论 |

等级映射建议：

| 子策略 action | 技术结构等级 |
|---|---|
| `can_sell_put` | A |
| `cautious` | B |
| `observe` | C |
| `pause` | D |

### 4.3 期权链参数判断文件

参考文件：

```text
策略_根据期权参数和期权链数据_判断每天卖什么Put分析SOP.md
```

作用：在市场风向和技术结构都允许的情况下，根据期权链截图或期权链参数，判断具体卖哪一个 Put。

#### 4.3.1 数据源口径：Barchart 为主

每日执行时，期权链和波动率数据优先使用 Barchart。

主口径：

```text
Barchart Options / Volatility & Greeks / IV Rank & Percentile
```

交叉验证：

```text
MarketChameleon / OptionCharts / AlphaQuery / Yahoo Finance / broker option chain
```

如果不同网站数据冲突：

```text
执行下单以 Barchart + 券商期权链为准；
其他网站只做温度计，不作为最终下单依据。
```

必须读取的字段：

- DTE；
- Strike；
- Bid / Ask / Mid；
- Delta；
- IV；
- HV / Historical Volatility；
- IV Rank；
- IV Percentile；
- Expected Move；
- IV 与历史波动率 HV 的关系；
- Volume；
- Open Interest；
- 安全垫；
- 盈亏平衡价；
- 年化收益率；
- 账户资金占用；
- 是否临近财报或重大事件；
- 是否与已有仓位风险因子重叠。

#### 4.3.2 IV 判断规则

```text
IV 高 ≠ 一定能卖。
IV 高 + IV Percentile 高 + IV Rank 高 + 技术结构没破 + 愿意接货 = 才值得考虑。
```

| 指标 | 判断 | 含义 |
|---|---|---|
| IV Percentile > 70% | 偏有利 | 过去一年多数时间 IV 低于当前水平，期权相对偏贵 |
| IV Rank > 50% | 偏有利 | 当前 IV 接近过去一年高位区间 |
| IV > HV | 有利 | 市场给的隐含波动高于实际波动，存在波动率溢价 |
| IV 高但 HV 更高 | 谨慎 | 真实波动也很大，不是白捡权利金 |
| Expected Move 下沿之外 | 有利 | Strike 更接近安全区域 |

IV 暴涨时的执行原则：

```text
平时卖收益，恐慌卖安全垫。
```

也就是说，IV 暴涨时不一定要卖更近的 Put，而应优先考虑卖更远 Strike，用高 IV 换更大的安全垫。

输出结果：

```text
期权链等级：A / B / C / D
```

| 等级 | 期权链状态 | 含义 | 操作 |
|---|---|---|---|
| A | 权利金合理、流动性好、安全垫足够 | 可以执行 | 正常考虑 |
| B | 基本可做，但有小瑕疵 | 谨慎执行 | 小仓或压低 Strike |
| C | 性价比一般 | 不急 | 等更好价格 |
| D | 流动性差或风险过高 | 不交易 | 放弃 |

### 4.4 波动率优先级规则

期权定价的核心是波动率。每日筛选候选 Put 时，必须把波动率作为期权链模块的核心输入之一。

基本原则：

```text
优先寻找 IV 较高、IV Rank / IV Percentile 较高、但市场风向和技术结构没有失控的标的。
```

每日寻找高权利金机会时，优先级必须明确：

```text
第一优先级：市场给错价的肥权利金 / 恐慌溢价
第二优先级：事件风险或暴雷风险带来的高权利金
```

第一优先级不是赌公司出事，而是寻找“市场把风险价格打得很高，但实际风险已经开始可控”的机会。典型特征是：IV、IV Rank、IV Percentile 偏高，Put/Call 或保护需求升温，但大盘环境没有恶化，个股技术结构没有破位，消息风险已经部分落地或正在降温。这类机会才是每日扫描的核心目标。

第二优先级才是事件风险、财报风险、监管风险、诉讼风险、融资风险、暴雷风险带来的高权利金。它们可能看起来权利金最肥，但本质上是在用高收益补偿真实尾部风险。除非市场风向、技术结构、事件路径、仓位承受能力都明确达标，否则默认只观察，不作为常规 Sell Put 主线。

高 IV 的意义是权利金更丰厚，但高 IV 可能来自两种完全不同的原因：

| 高 IV 来源 | 含义 | 卖 Put 动作 |
|---|---|---|
| 市场恐慌后趋稳、波动率被高估 | 权利金补偿风险，卖方可能有优势 | 第一优先级，可进入候选 |
| 财报、诉讼、监管、破位、流动性危机、暴雷风险 | 权利金补偿真实尾部风险 | 第二优先级，默认降级或暂停 |

因此不能只看绝对 IV。必须同时看：

| 指标 | 用途 |
|---|---|
| IV | 判断当前权利金是否丰厚 |
| IV Rank | 判断当前 IV 在过去一年区间中的相对位置 |
| IV Percentile | 判断当前 IV 高于过去多少交易日 |
| IV / HV | 判断期权隐含波动是否高于实际历史波动 |
| Put/Call Ratio | 判断保护需求是否异常 |
| Bid / Ask Spread | 判断高 IV 是否真的可成交 |
| OI / Volume | 判断该档 Put 是否有流动性 |

优先级：

```text
IV Rank / IV Percentile 高
+ 技术结构 A/B
+ 市场风向 A/B
+ Bid/Ask 合理
+ OI/Volume 合格
= 高质量卖 Put 候选
```

每日扫描顺序：

1. 先找“波动率被高估但风险可控”的标的。
2. 再找“恐慌溢价正在回落但权利金仍然肥”的标的。
3. 最后才研究“事件风险 / 暴雷风险给出的极高权利金”。
4. 如果当天只有暴雷型高 IV，没有可控错价机会，则宁可不交易。

降级规则：

- IV 高但技术结构 D：不交易；
- IV 高但市场风向 C/D：不交易；
- IV 高但 Bid/Ask 过宽：不交易；
- IV 高但临近财报或重大事件：默认降级或暂停；
- IV 高主要来自暴雷风险、监管风险、诉讼风险或融资断裂风险：默认不作为第一候选；
- IV 高但 Strike 不是自己愿意接货的位置：不交易；
- IV 已经大幅回落，权利金不足：不勉强卖。

一句话：

```text
卖 Put 要卖“被高估的波动率”，不是卖“正在爆炸的风险”。
```

## 五、评分与最终决策规则

最终操作等级取三关中的最低等级：

```text
最终等级 = 市场风向等级、技术结构等级、期权链等级中的最低等级
```

| 市场风向 | 技术结构 | 期权链 | 最终操作 |
|---|---|---|---|
| A | A | A | 可以卖 Put |
| A | B | A | 可以小仓卖 Put |
| A | C | A | 暂缓，等技术确认 |
| B | A | A | 小仓、远虚值 |
| C | A | A | 不卖 |
| D | A | A | 不卖，等事件落地 |
| A | A | C | 不急，等更好价格 |
| A | A | D | 不交易 |
| B | B | B | 可小仓试单，但必须降低 Delta |
| A | D | A | 不卖，技术破位不能用权利金补偿 |

## 六、Delta 使用规则

期权链分析中，Delta 分为两种口径：

```text
比较 Delta：用于横向比较不同标的的期权性价比。
执行 Delta：用于真实下单，需要根据标的风险和市场风向调整。
```

### 6.1 比较 Delta

默认使用接近：

```text
|Delta| ≈ 0.15
```

用于横向比较：

- 哪个标的权利金更合理；
- 哪个标的安全垫更充足；
- 哪个标的年化收益更高；
- 哪个标的风险补偿更合理。

### 6.2 执行 Delta

真实下单时，不同标的不能机械使用同一个 Delta。

| 标的 | 顺风环境 | 横风环境 | 逆风/事件风 |
|---|---|---|---|
| QLD | 0.10–0.20 | 0.08–0.12 | 不开新仓 |
| EEM | 0.10–0.15 | 0.08–0.12 | 不开新仓 |
| MSTR | 0.05–0.10 | 0.03–0.06 | 不开新仓 |
| INTC | 0.10–0.15 | 0.08–0.10 | 不开新仓 |
| MARA / RIOT / IBIT | 0.05–0.10 | 0.03–0.06 | 不开新仓 |

原则：

```text
波动越大，Delta 越低。
事件越近，Delta 越低。
市场越逆风，越不应该新开仓。
ETF 可以比高波动个股略高，但不能因为“ETF”就忽略宏观风险。
```

## 七、DTE 使用规则

默认优先观察：

```text
7–14 DTE：短线周度 Sell Put
21–45 DTE：月度主线 Sell Put
90–180 DTE：远期低价接货型 Put，仅限愿意长期接货的标的
```

### 7.1 周度 Sell Put

适合条件：

- 市场风向 A 或 B；
- 技术结构 A 或 B；
- 标的流动性好；
- 期权价差合理；
- 没有重大事件。

特点：收益快、容易滚动，但对短期波动更敏感。

### 7.2 月度 Sell Put

适合条件：

- 有较大安全垫；
- 权利金明显比周度更合理；
- 用户愿意接货；
- 账户资金占用可以接受。

特点：年化更容易稳定，时间缓冲更大，但持仓周期更长。

### 7.3 远期 Put

90–180 DTE 的 Put 不作为现金流主策略，而作为“带权利金的限价买入单”。

适合：QQQ / QLD / EEM 这类愿意长期接货的 ETF，或用户明确愿意接货的核心标的。

不适合：MSTR / MARA / RIOT / IONQ 等高波动个股重仓远期卖 Put；CRCL 固定排除。

## 八、仓位与风险因子规则

### 8.1 单标的仓位

单一标的不能因为权利金高而过度集中。

| 标的 | 单次新开仓建议 |
|---|---|
| QLD | 可作为主力，但需要控制总张数 |
| EEM | 稳健 ETF 候选，小到中等仓位；不追近价，不在美元强势时硬做 |
| MSTR | 小仓，不能和 BTC / IBIT / MARA / RIOT 风险过度叠加 |
| INTC | 事件过滤后小到中等仓位 |
| MARA / RIOT / IBIT | 实验小仓，不能和 MSTR 风险过度叠加 |

### 8.1.1 2 万美元账户适配规则

本策略默认服务 2 万美元左右的小账户，所有候选 Put 必须先通过账户适配检查，再比较年化收益率。

```text
优先目标：用较小接货金额获得合理波动率溢价，而不是用高价标的占满账户。
```

2 万美元账户硬约束：

| 项目 | 规则 |
|---|---|
| 单张接货金额 | 优先低于 5,000 美元；5,000–8,000 美元为中等占用；超过 10,000 美元必须小仓或放弃 |
| 单笔最大占用 | 原则上不超过账户 50%，除非是高流动性 ETF 且市场顺风 |
| 同时风险暴露 | 同一风险因子最多保留 1 个主仓，不叠加多个同向高 beta 仓位 |
| 高波动个股 | MSTR、MARA、RIOT、IONQ、SMCI 只能实验小仓，不能作为账户主仓 |
| ETF 优先级 | 同等年化下，优先 ETF 或接货金额更低的标的 |
| 保留现金 | 开仓后必须保留足够现金，避免一次接货导致账户失去调整空间 |

2 万账户优先筛选顺序：

```text
第一优先：接货金额 1,500–5,000 美元、DTE 5–10 天、Delta 0.12–0.18、年化 >= 40%、安全垫 >= 7%
第二优先：接货金额 5,000–8,000 美元、ETF 或结构稳定标的、年化明显优于同类
第三优先：接货金额 8,000–10,000 美元，只能在强顺风、小仓、愿意接货时考虑
默认放弃：单张接货金额超过 10,000 美元且不是极佳机会
```

如果一个 Put 年化达到 40%，但单张接货金额过大、风险因子与已有仓位重叠，或者接货后账户失去调整空间，默认降级为“不适合”。

### 8.2 风险因子重叠

以下标的或资产具有较高风险相关性，不能同时重仓：

| 风险因子 | 相关标的 |
|---|---|
| Nasdaq / 科技股 | QLD、QQQ、NVDA、MSFT、AAPL、AMZN、META、GOOG |
| AI / 半导体 | QLD、INTC、SMH、SOXX、NVDA、AMD、AVGO |
| BTC / Crypto | MSTR、MARA、RIOT、IBIT、BTC |
| 美元 / 新兴市场 | EEM、FXI、KWEB、亚洲市场、资源国资产 |
| 利率敏感资产 | QLD、成长股、科技股、长久期资产、EEM |

如果已有仓位已经暴露在某个风险因子上，新开仓必须降低张数、降低 Delta 或放弃交易。

## 九、一票否决规则

只要出现以下任意情况，即使权利金很高，也不主动卖 Put：

1. 市场风向为 C 或 D；
2. VIX 快速上升，市场进入风险释放期；
3. 个股或 ETF 放量跌破关键支撑；
4. 标的跌破 ABC 结构中的关键低点；
5. 股价跌破 50 日线后没有修复；
6. 财报、监管、诉讼、政策等重大事件未落地；
7. Put 权利金突然变肥，但同时股价破位；
8. Bid / Ask 价差过大；
9. Open Interest 太低；
10. 账户资金占用超过计划上限；
11. 同一风险因子已经暴露过高；
12. 用户主观上不愿意按 Strike 接货；
13. 只能靠“我觉得应该会反弹”来解释交易；
14. 年化收益很高，但安全垫不足；
15. 标的进入暂停池；
16. EEM 遇到 DXY 快速上行、10Y 快速冲高、新兴市场风险释放时，不主动卖 Put。

一句话：

```text
权利金不能补偿结构性破位。
```

## 十、每日自动执行入口

以后每日执行本总控策略时，优先使用自动化入口准备基础数据，不再默认依赖手工上传 Barchart 截图和期权链截图。

自动化入口的目标不是自动下单，而是先完成：

```text
自动抓取市场和期权基础数据
↓
生成每日 Sell Put 数据包
↓
生成符合本总控模板的日报骨架
↓
由 AI 按市场风向 + 技术结构 + 期权链三关输出结论
```

### 10.1 每日执行命令

在本地项目目录 `D:\codex\金融` 下执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_daily_sell_put_data.ps1 -Symbols "QLD,EEM,TQQQ,GDX,ARKK,IBIT,SOFI,RIVN,SMCI,INTC,MSTR,MARA,RIOT,IONQ,HOOD,PLTR" -OptionChainLimit 300 -Jin10SearchType "flash" -Jin10WindowMode "SellPut"
```

默认扫描池：

```text
QLD / EEM / TQQQ / GDX / ARKK / IBIT /
SOFI / RIVN / SMCI / INTC / MSTR / MARA / RIOT / IONQ / HOOD / PLTR
```

固定排除：

```text
CRCL
```

如当天只想扫描部分标的，可以修改 `-Symbols` 参数，例如：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_daily_sell_put_data.ps1 -Symbols "QLD,MSTR" -OptionChainLimit 200
```

### 10.2 自动生成的数据文件

每日自动入口应生成以下三类文件：

| 文件 | 用途 |
|---|---|
| `jin10_market_news_YYYY-MM-DD.csv` | 抓取金十搜索 `type=flash` 的市场新闻输入，按 Sell Put 交易窗口过滤 |

这些文件主要替代过去手工提供的：

- 金十市场新闻链接和手工摘录；
- Barchart Options Overview 截图；

金十市场新闻输入固定使用以下口径：

```text
搜索类型：type=flash
关键词：金十数据整理
时间过滤：SellPut 交易窗口
```

SellPut 交易窗口规则：

```text
周二到周五：抓取从昨天晚上 21:00 到当前时间的数据
周六、周日、周一：抓取从周五晚上 20:00 到当前时间的数据
```

这样做的原因是，美股 Sell Put 判断主要服务美股盘前、盘中和隔夜风险。周二到周五需要覆盖前一晚美股交易时段之后到当前的新闻；周末和周一需要覆盖从周五晚间美股风险窗口开始的全部周末新闻。

不再默认纳入交易窗口之前的早盘、午后、债券、地产、A 股等较早整理稿，除非用户明确要求扩大时间范围。

### 10.4 每日 AI 判断流程

当自动数据包生成后，AI 必须按以下顺序输出结论：

```text
第一步：读取每日数据包
第二步：按市场风向研究模式判断市场风向等级
第三步：单独检查日元 carry trade 平仓风险，并输出“无明显苗头 / 有苗头 / 已触发降级”
第四步：按动量与 ABC 策略判断技术结构等级
第五步：按期权链 SOP 判断期权链等级
第六步：三关取最低等级，并叠加日元 Carry 风险降级规则，得到最终操作等级
第七步：输出“适合 / 小仓适合 / 不适合”
第八步：写明候选 Put、放弃原因、失效条件和复盘要点
```

AI 不得因为自动数据包中某个标的权利金较高，就跳过市场风向和技术结构判断。

## 十三、交易纪律

1. 不因为权利金高而卖 Put。
2. 不因为年化收益率高而忽略安全垫。
3. 不在市场逆风时为了交易而交易。
4. 不在技术破位时用“应该会反弹”安慰自己。
5. 不在重大事件前重仓卖 Put。
6. 不在流动性差的期权链里勉强成交。
7. 不让单一风险因子占用过多账户资金。
8. 不卖自己不愿意接货的 Strike。
9. 不把观察池标的临时变成交易标的。
10. 不让期权链反过来决定股票池。
11. EEM 只能作为稳健 ETF 候选，不能因为 IV Percentile 高就追近价。
12. IV 暴涨时优先增加安全垫，而不是提高贪婪程度。
13. Barchart 是执行口径，其他网站是辅助温度计。

## 十五、最终原则

本策略的核心不是每天都要交易，而是等到赔率合适时出手。

```text
市场顺风时，卖 Put 是收租。
市场逆风时，卖 Put 是接刀。
IV 暴涨时，不是更贪，而是换更大安全垫。
```

每天最终只需要回答一个问题：

```text
今天这个 Put，是我愿意接货的好价格，还是市场用高权利金诱惑我去承担不该承担的风险？
```

如果答案不清楚，就不交易。
