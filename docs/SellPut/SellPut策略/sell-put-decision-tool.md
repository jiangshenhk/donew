# Sell Put 综合决策工具 — 设计思路

## 1. 为什么需要这个工具

卖 Put 不是单一维度的判断。好的卖 Put 决策需要同时考虑四个维度：

```
                    ┌─ 市场环境（大盘情绪、VIX、利率、地缘）
                    ├─ 新闻事件（财报、FOMC、非农等近期风险）
卖Put综合判断 ──────┼─ 技术面（K线趋势、ATR波动、支撑阻力）
                    └─ 期权温度（IV/HV溢价、恐慌程度、权利金价值）
```

原来的四个工具**各自独立**，用户需要手动在不同页面间切换、复制结论、脑补关联。这个工具的目标是**一次生成，四维合一**。

## 2. 核心设计原则

### 2.1 数据维度不是越多越好，是越结构化越好

不是把四个数据源扔进 prompt 就完事。每个数据源需要：
- **结构化提取**：新闻扫描事件风险、K线计算技术指标、行情快照算风险分
- **量化结论**：不是"感觉不对"，而是给出具体的数字（风险分、ATR%、安全垫%）
- **可追溯**：每个结论后面都有数据锚点

### 2.2 代码判断 + AI 综合，两层决策

```
代码层（规则引擎）                      AI 层（综合判断）
┌─────────────────────┐          ┌──────────────────────┐
│ 市场风险评分（大盘）  │          │ 1. 解读评分含义       │
│ + 单票风险调整（K线） │  ──→    │ 2. 综合四个维度       │
│ + 事件风险扫描（新闻） │          │ 3. 生成格式化报告     │
└─────────────────────┘          └──────────────────────┘
```

**规则引擎负责**：客观计算、不可被 AI 跳过
- 市场风险评分（VIX、QQQ、10Y、DXY 等大盘指标）
- 单票风险调整（ATR%、20 日趋势、5 日趋势）
- 事件关键词扫描

**AI 负责**：解读、综合、表达
- 读评分背后的含义（不是念分数）
- 结合四个维度的信息给出判断
- 生成结构化的可视化报告
- 利用训练知识补充事件风险（财报/FOMC 日历等代码无法获取的信息）

### 2.3 风险评分是硬约束，不是建议

三层风险判定：

| 风险分范围 | 盘面判定 | 含义 | 代码强制 |
|---|---|---|---|
| < 6.2 | 可卖Put | 市场平稳，溢价真实 | 否 |
| 6.2 - 7.5 | 谨慎卖Put | 有风险信号，需减仓 | 否 |
| ≥ 7.5 | 暂不卖Put | 风险大于收益 | 否（AI可覆盖） |

风险分通过 Hero 区域 chip 展示，AI prompt 也收到调整依据，AI 可以基于更全面的信息做微调，但代码层的风险分是强信号。

## 3. 评分体系详解

### 3.1 第一层：市场环境评分（`marketRisk`）

基准分 **5.2**。以下事件触发加分（分数越高 = 风险越大）：

| 大盘信号 | 加分 | 含义 |
|---|---|---|
| VIX 日涨 >5% | +1.2 | 恐慌情绪飙升 |
| 10Y 收益率涨 >1% | +0.7 | 利率压制科技股 |
| DXY 涨 >0.3% | +0.6 | 美元走强，风险资产承压 |
| QQQ / SPY 跌 >1% | +0.9 | 大盘明显走弱 |
| SMH / SOXX 跌 >1% | +0.7 | 半导体领跌 |
| BTC 跌 >2.5% | +0.5 | 风险偏好恶化 |
| IWM 跑赢 SPY | -0.2 | 防御轮动（其实是好事） |
| 标的自身跌 >3% | +0.6 | 个股加速下跌 |

### 3.2 第二层：单票 K线调整（`adjustedRisk`）

大盘评分只反映宏观环境。个股自身的趋势和波动也需要纳入：

| K线信号 | 加分 | 适用场景 |
|---|---|---|
| ATR 占比 >8% | +1.0 | 极度波动（如 INTC） |
| ATR 占比 >6% | +0.5 | 波动偏高 |
| 20 日跌幅 >20% | +1.0 | 自由落体式下跌 |
| 20 日跌幅 >10% | +0.5 | 持续走弱 |
| 5 日跌幅 >5% | +0.3 | 短线加速下跌 |

**为什么不减分（利好不加分）？**
卖 Put 是风险定价工具，我们的目标是识别风险，不是找"完美时机"。利空信号需要拉响警报，利多信号保持默认为好。

### 3.3 调整示例

INTC（2026-07-20）：
```
原始评分: 5.2 + 0.7(10Y↑) = 5.9 → "可卖Put"
  +1.0 (ATR 9.18% >> 8%)
  +1.0 (20日跌 26.8% >> 20%)
  +0.3 (5日跌 5.2% > 5%)
= 8.2 → "暂不卖Put"
```

## 4. 数据流架构

```text
用户输入: symbol / screenshot / metrics / notes
  │
  ├─→ Promise.all 并行拉取 ─────────────────────┐
  │   ├─ stockprice/data/latest-price.json      │  行情快照
  │   ├─ jin10news/data/latest-24h.json         │  72h 新闻
  │   └─ Yahoo Finance chart API                │  60 日K线
  │                                              │
  ├─→ 代码层处理 ─────────────────────────────┐  │
  │   ├─ computeKlineStats()   ← K线原始数据   │  │
  │   │   ├─ SMA5/10/20/50                    │  │
  │   │   ├─ ATR(14)                          │  │
  │   │   ├─ 形态检测（十字/吞没/锤子等）        │  │
  │   │   ├─ 支撑/阻力位                       │  │
  │   │   └─ 1/5/20日涨跌幅                    │  │
  │   ├─ marketRisk()         ← 行情快照        │  │
  │   ├─ adjustedRisk()       ← 风险分+K线      │  │
  │   ├─ scanEventRisks()     ← 新闻            │  │
  │   └─ OCR (OpenAI Vision)  ← 截图            │  │
  │                                              │
  └─→ AI Prompt ───────────────────────────────┘
      ├─ 结构化数据注入（不依赖 AI 提取）
      ├─ 风险判断铁律（三问必须答）
      ├─ 六节报告格式（综合结论到关注清单）
      └─ 视觉风格规范（badge/tag/data-item/bullet-list）
```

## 5. 输出报告结构

```
<section class="hero-judgement">   ← 综合结论（三问+事件风险+量化数据）
<section class="section">          ← 市场环境（大盘 data-item + bullet-list）
<section class="section">          ← 期权温度解读（tag + highlight + bullet-list）
<section class="section">          ← K线技术信号（flex-between + table + bullet-list）
<section class="section">          ← 综合卖Put建议（action bar + 编号风险 + 警告框）
<section class="section">          ← 未来关注清单（两列信号 + 改变判断标签）
<details>                          ← 实时行情快照表格
```

## 6. 与其他工具的关系

这不是替换，是**聚合层**，骑在现有基础设施之上：

- `stockprice/` → 行情底座（数据同上）
- `jin10news/` → 新闻底座（数据同上）
- 独立页面 → 用户仍然可以单独使用原有三个工具
- 综合决策 → 新增的"一键生成"入口
