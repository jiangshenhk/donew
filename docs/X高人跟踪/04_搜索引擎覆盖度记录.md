# X 高人库：搜索引擎覆盖度记录

> 目的：在没有完整 X API/时间线抓取能力的情况下，先评估每个账号通过公开搜索引擎能查到多少历史和最新发言，避免把“搜不到”误判为“没有发言”。

## 一、记录字段

| 字段 | 说明 |
|---|---|
| 账号 | X 用户名与显示名 |
| 检索日期 | 本轮搜索日期 |
| 查询组合数 | 使用了多少组“账号 + 关键词 + 时间”查询 |
| 可辨认原帖数 | 能确认是该账号原始 X 发言的数量 |
| 第三方引用数 | 媒体、博客、其他账号对其观点的引用数量 |
| 最早可见日期 | 本轮能找到的最早发言时间 |
| 最新可见日期 | 本轮能找到的最新发言时间 |
| 覆盖等级 | 高 / 中 / 低 / 几乎不可见 |
| 能否做准确率回测 | 可以 / 仅抽样 / 不可以 |
| 备注 | 回复帖、删帖、图片帖、同步平台等缺失情况 |

## 二、覆盖等级

- **高**：可找到 50 条以上原帖，且最近 7 天内容基本可见。
- **中**：可找到 20–49 条原帖，最近 30 天有持续结果。
- **低**：只能找到 5–19 条零散原帖或大量第三方引用。
- **几乎不可见**：少于 5 条可确认原帖。

## 四、使用规则

1. 每次人物审计必须同时报告“找到多少条”，不能只报告命中案例。
2. 搜索结果按“原帖、第三方引用、同步平台”分开计数。
3. 最新观点必须注明来源层级：X 原帖 > 同步平台 > 权威媒体引用 > 普通转述。
4. 搜索覆盖低时，只给“抽样判断”，不计算正式准确率。
5. 后续如使用 Playwright、twscrape 或官方 API，应另行记录真实采集条数，与搜索引擎结果对照。
