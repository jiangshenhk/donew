# X 高人库：全量历史审计台账

> 目标：覆盖候选账号的历史公开发言，提取可验证观点，分别评价预测能力、信息价值和交易可用性。没有达到最低样本量前，不给出正式准确率。

最后更新：2026-07-12

## 一、审计口径

### 1. 历史覆盖

每个账号记录：

- 最早可获得发言日期
- 最晚可获得发言日期
- 已采集原创帖数量
- 已采集转发/引用数量
- 含明确投资判断的帖子数量
- 可被行情验证的预测数量
- 公开历史缺失程度

### 2. 正式评级最低要求

- 至少采集 100 条原创投资相关发言，或覆盖最近 12 个月；
- 至少提取 20 条可验证预测；
- 正确和错误观点必须同时保存；
- 短线、波段、周期判断分开统计；
- 只有方向但没有时间窗口的观点，不能计入严格准确率；
- 最终碰到目标但中途先发生巨大反向波动，标记为“方向命中、交易不可用”。

### 3. 三种价值分开评价

| 类型 | 说明 |
|---|---|
| 预测价值 | 对方向、价位和时点的判断能力 |
| 信息价值 | 能否持续提供链上、衍生品、宏观或行业一手信息 |
| 框架价值 | 是否提供稳定、可复用的投资方法，而非单次喊单 |

## 三、采集与回测输出

每个人最终形成四份内容：

1. **人物档案**：身份、市场、方法、利益关系、营销方式；
2. **历史预测表**：原帖、发布时间、当时价格、方向、目标、时间、失效位、结果；
3. **周度/月度观点快照**：当前立场及相对上期变化；
4. **最终判定**：适合用于预测、信息、框架，还是应淘汰。

## 四、当前数据限制

X 个人主页的公开网页并不稳定提供完整历史时间线；搜索引擎也经常只索引少量热门或被引用的帖子。因此，不能把网页搜索命中的少数帖子视作该账号的完整历史。

正式审计需要使用以下任一来源：

- X 登录状态下导出的个人时间线；
- 用户提供的 X 数据归档或批量导出文件；
- 合规的 X API 数据；
- 从浏览器中按时间滚动采集的原帖数据。

在完成原始语料采集前，除 TraderChenge 的阶段性抽样和 Ray Dalio 的框架评级外，其余账号均保持“观察中”。
